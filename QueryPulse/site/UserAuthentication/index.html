    User Management¶

    Authentication¶

    Query Pulse supports three authentication modes, configured via the QUERY_PULSE_AUTH_TYPE environment variable:

    Mode Description User Provisioning
    local (default) Users register and authenticate with a username and password stored in the application database Manual registration
    ldap Users authenticate against an LDAP or Active Directory server Users must be pre-registered locally
    oidc Users authenticate via an OpenID Connect Identity Provider (SSO) Automatic on first login (configurable)

    All modes use JWT (JSON Web Token) based stateless authentication for API requests:

    • Login: Users authenticate via their configured method
    • Token Refresh: Access tokens can be refreshed without re-authentication
    • Session Management: Tokens are stored client-side; logout clears local storage

    Local Authentication¶

    The default mode. Users register with a username, email, and password. Passwords are stored as bcrypt hashes in the local database.

    LDAP Authentication¶

    Delegates credential verification to an external LDAP or Active Directory server. Users must be pre-registered in Query Pulse with a username matching their directory entry. See LDAP Authentication for setup instructions.

    OIDC / SSO Authentication¶

    Enables Single Sign-On via an OpenID Connect provider (Keycloak, Okta, Azure AD, etc.). Users are automatically provisioned on first login. See OIDC / SSO Authentication for setup instructions.

    Admin Account¶

    Regardless of the authentication mode, the admin account (default username: admin) always authenticates using local credentials. This ensures admin access is maintained even when external authentication services are unavailable.

    Admin vs Non-Admin Users¶

    The application distinguishes between two user types:

    Capability Admin User Regular User
    Manage own connections Yes Yes
    Manage own Kerberos configs Yes Yes
    Manage own Git credentials Yes Yes
    Upload certificates Yes Yes
    View system status Yes Yes
    Manage JDBC drivers Yes No
    Configure TLS/mTLS Yes No
    Manage RocksDB Yes No
    Configure JVM settings Yes No
    Configure LLM providers Yes No
    Rotate encryption keys Yes No
    Manage other users Yes No
    Create global certificates Yes No

    Admin Role Assignment:

    • Local mode: The first registered user automatically receives the ADMIN role. Additional admins can be promoted by existing admins.
    • LDAP mode: Admin role is managed manually through the Query Pulse user management interface.
    • OIDC mode: Admin role can be automatically assigned based on IdP group membership via QUERY_PULSE_OIDC_ADMIN_GROUP.
