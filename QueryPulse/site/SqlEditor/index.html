    SQL Editor¶

    Query Pulse provides a powerful SQL editing environment designed for data engineers and analysts working with LakeHouse platforms. The editor supports multiple SQL dialects, real-time query progress, and comprehensive result management.

    Editor Overview¶

    The SQL Editor is the primary workspace for writing and executing SQL queries. It combines a modern code editor with database connectivity, variable substitution, and rich result visualization.

    Multi-Tab Sessions¶

    Work with multiple database connections simultaneously:

    • Each tab maintains its own session and connection context
    • Switch between connections without losing previous results
    • Execute queries in parallel across different databases
    • Sessions preserve state when switching between file tabs

    SQL Dialect Support¶

    The editor supports multiple SQL dialects with dialect-specific features:

    Dialect Features
    Hive Full Hive SQL, EXPLAIN variants (Visual, CBO, Dependency, Authorization, Vectorization)
    Trino Trino SQL, EXPLAIN with TYPE and FORMAT options
    Impala Coming soon

    Writing SQL¶

    Syntax Highlighting¶

    The editor provides real-time syntax highlighting for SQL keywords, strings, numbers, and comments. Highlighting adapts automatically to your chosen theme (Day/Night).

    Code Formatting¶

    Format your SQL for better readability:

    1. Format Selection: Select text and format only that portion
    2. Format All: Format the entire query

    Two formatters are available:

    • T-SQL Formatter: Fast, client-side formatting (always available)
    • Calcite Formatter: Advanced formatting via the backend (preferred for DML)

    Access formatting via the toolbar or right-click context menu.

    Code Assist¶

    The editor provides intelligent autocompletion for:

    • SQL keywords and functions
    • Table and column names from your schema
    • Database/schema names

    For detailed information on Code Assist features, see the Code Assist Guide.

    Find and Replace¶

    • Find: Ctrl+F (Windows/Linux) or Cmd+F (Mac)
    • Replace: Ctrl+R (Windows/Linux) or Cmd+R (Mac)

    Features: - Case-sensitive search option - Highlight all matches - Navigate between matches with arrow buttons

    Executing Queries¶

    Execution Modes¶

    Mode Shortcut (Win/Linux) Shortcut (Mac) Description
    Run Statement Ctrl+Enter Cmd+Enter Execute the statement at cursor position
    Run Script Ctrl+Shift+Enter Cmd+Shift+Enter Execute all statements sequentially
    Run from Cursor Ctrl+Alt+Enter Cmd+Alt+Enter Execute from cursor to end of file

    Statement Detection¶

    The editor automatically detects statement boundaries:

    • Statements are separated by semicolons (;)
    • Comments and quoted strings are properly handled
    • Leading comments can serve as statement labels

    Cancel Query¶

    Click the Cancel button to stop a running query. The editor shows "Cancelling..." status during the cancellation process.

    Variable Substitution¶

    Use variables to parameterize your queries and avoid hardcoding values.

    Defining Variables¶

    Variables can be defined at two levels:

    1. Workspace Variables: Shared across all files in the workspace
    2. File Variables: Override or extend workspace variables for a specific file

    Using Variables¶

    Reference variables in your SQL using the ${variable_name} syntax:

    SELECT *
FROM ${database}.${table}
WHERE date = '${run_date}'
LIMIT ${max_rows}

    Missing Variables¶

    If your query contains undefined variables:

    1. Execution is paused
    2. A dialog prompts you to define missing variables
    3. Once defined, execution continues

    File Variables Panel¶

    The right sidebar includes a Variables panel where you can:

    • View all available variables (workspace + file)
    • Add file-level variables
    • Override workspace variables
    • Delete file-level overrides

    Query Results¶

    Result Tabs¶

    After execution, results appear in tabs below the editor:

    Tab Content
    Results Query output as a data table
    Statements History of executed statements with timing
    Details Query metadata, statistics, and operator info
    Progress Live execution progress (during query run)

    Table View¶

    The default result display with:

    • Column headers with data types
    • Sortable columns
    • Row numbers
    • Scrollable content

    Filtering Results¶

    Filter displayed rows using the filter box:

    Pattern Matches
    text Rows containing "text" (case-insensitive)
    ^prefix Rows starting with "prefix"
    suffix$ Rows ending with "suffix"
    ^exact$ Exact match only
    /regex/ Regular expression pattern

    The filter status shows: Showing X of Y rows

    Pagination¶

    Navigate large result sets:

    • Page Size: Default 25 rows per page (configurable in preferences)
    • Navigation: Previous/Next buttons and direct page input
    • Range Display: Shows current row range (e.g., "1-25 of 1,000")

    Raw View¶

    Switch to Raw View for tab-separated output, useful for:

    • Copy/paste into spreadsheets
    • Text processing
    • Quick data inspection

    Export Options¶

    Export results in multiple formats:

    Format Description
    CSV Comma-separated values
    TSV Tab-separated values
    Markdown Markdown table format
    Excel XLSX spreadsheet

    Exports respect the current filter, exporting only visible rows.

    Query Progress¶

    Real-Time Tracking¶

    During query execution, view live progress:

    • Overall completion percentage
    • Stage-by-stage breakdown
    • Elapsed time (mm:ss)
    • Task counters: Running, Completed, Pending, Failed

    View Modes¶

    • Summary: High-level progress overview
    • Detailed: Per-stage task breakdown

    Platform-Specific Metrics¶

    Hive/Tez: - DAG progress - Vertex status - Task completion counts

    Trino: - Query state (QUEUED, RUNNING, FINISHED) - CPU and memory usage - Data processing statistics - Execution time breakdown

    Explain Plans¶

    Visualize query execution plans without running the query.

    Hive EXPLAIN Options¶

    Option Description
    Visual Explain Graphical DAG representation
    Visual Cost Cost-based optimization plan
    Explain Dependency Table and column dependencies (JSON)
    Explain Authorization Required permissions
    Explain Vectorization Vectorized execution plan

    Trino EXPLAIN Options¶

    Type Description
    LOGICAL Logical execution plan
    DISTRIBUTED Distributed plan with stages
    IO I/O operations plan
    VALIDATE Syntax validation only

    Output formats: TEXT, GRAPHVIZ (visual), JSON

    Viewing Plans¶

    • Visual plans open in the DOT Graph Editor
    • JSON/text output opens in a new file tab
    • Plans are versioned for comparison

    Keyboard Shortcuts¶

    Query Execution¶

    Action Windows/Linux Mac
    Run Statement Ctrl+Enter Cmd+Enter
    Run Script Ctrl+Shift+Enter Cmd+Shift+Enter
    Run from Cursor Ctrl+Alt+Enter Cmd+Alt+Enter

    File Operations¶

    Action Windows/Linux Mac
    Save Ctrl+S Cmd+S
    Find Ctrl+F Cmd+F
    Replace Ctrl+R Cmd+R

    Editor¶

    Action Windows/Linux Mac
    Undo Ctrl+Z Cmd+Z
    Redo Ctrl+Y Cmd+Shift+Z
    Select All Ctrl+A Cmd+A

    Theme Support¶

    The SQL Editor adapts to your theme preference:

    • Day Theme: Light background with dark text (GitHub Light)
    • Night Theme: Dark background with light text (GitHub Dark)
    • Auto: Follows your system preference

    Change themes in Settings > Preferences > Theme.

    Other Editors¶

    Query Pulse includes specialized editors for different file types:

    Code Editor (YAML, JSON, XML)¶

    For configuration and data files:

    • Syntax highlighting per language
    • Save/dirty state tracking
    • Git integration

    DOT Graph Editor (.dot files)¶

    For visualizing execution plans and graphs:

    • Source View: Edit DOT code directly
    • Split View: Editor and preview side-by-side
    • Graph View: Full-screen rendered graph
    • Zoom controls (in, out, fit to view)
    • Live preview updates

    Markdown Editor (.md files)¶

    For documentation:

    • Source View: Edit Markdown
    • Split View: Editor and preview
    • Preview View: Rendered Markdown
    • Syntax highlighting in code blocks
    • Live preview

    Tips¶

    Efficient Query Development¶

    1. Use variables for values that change between runs
    2. Format your SQL before committing to version control
    3. Use Run Statement to test individual queries in a script
    4. Check EXPLAIN before running expensive queries

    Working with Large Results¶

    1. Filter early in your SQL with WHERE clauses
    2. Use LIMIT during development
    3. Export filtered results to reduce file size
    4. Use pagination to navigate without loading all rows

    Connection Management¶

    1. Create connections for each database you work with
    2. Set workspace defaults for commonly used connections
    3. Override at file level when needed for specific scripts
