    Standalone Installation¶

    Install Query Pulse directly on a Linux server or edge node.

    Prerequisites¶

    • Java JDK 17 or later
    • 4 GB RAM minimum, 8 GB recommended

    Installation¶

    Download the Query Pulse release archive (query-pulse-install-<version>-dist.tar.gz).

    Step 1 — Extract the archive:

    tar zxvf query-pulse-install-1.3.0-dist.tar.gz
cd query-pulse-install-1.3.0

    The archive contains:

    query-pulse-install-1.3.0/
  setup.sh           # Installation script
  web.properties     # Default configuration
  bin/
    query-pulse.sh   # Startup script
  lib/
    query-pulse-1.3.0.jar

    Step 2 — Run setup:

    ./setup.sh

    The setup script validates your Java version (17+ required), then installs the application. The install location depends on whether you run as root or a regular user:

    Run As Application Binaries Data & Configuration
    Regular user $HOME/.query-pulse/bin, lib $HOME/.query-pulse/conf, data, logs
    Root /usr/local/query-pulse/bin, lib $HOME/.query-pulse/conf, data, logs

    When run as root, a symlink is created at /usr/local/bin/query-pulse. When run as a regular user, the symlink goes to $HOME/bin/query-pulse — add $HOME/bin to your PATH if it isn't already.

    The setup preserves any existing web.properties in $HOME/.query-pulse/conf/, so re-running setup for an upgrade will not overwrite your configuration.

    Step 3 — Start the application:

    query-pulse start

    Verify: Open http://<hostname>:8088 in your browser.

    Managing the Service¶

    query-pulse start     # Start in the background
query-pulse stop      # Stop the service
query-pulse status    # Check if running

    Configuration¶

    Query Pulse uses two properties files, both managed through the admin UI under Settings > Administration. Changes to these files require a restart to take effect.

    File Location Purpose
    web.properties $HOME/.query-pulse/settings/web.properties JVM, logging, authentication, and application settings
    tls.properties $HOME/.query-pulse/settings/tls.properties TLS/mTLS and certificate settings

    Settings can also be overridden with environment variables:

    export QUERY_PULSE_SERVER_PORT=8088        # HTTP port (default: 8088)
export QUERY_PULSE_HEAP_INIT=2048          # Initial heap in MB
export QUERY_PULSE_HEAP_MAX=4096           # Max heap in MB
export QUERY_PULSE_LOG_LEVEL=INFO          # DEBUG, INFO, WARN, ERROR

    Directory Structure¶

    After installation and first run, the working directory contains:

    $HOME/.query-pulse/
  bin/                # query-pulse.sh
  lib/                # Application JAR
  conf/               # web.properties (initial copy)
  settings/           # web.properties, tls.properties (managed by the app)
  data/               # RocksDB embedded database
  drivers/            # JDBC driver JARs
  security/           # Keystores and certificates
  logs/               # Application logs
  run/                # PID file

    Upgrading¶

    To upgrade to a new version:

    1. Stop the running instance: query-pulse stop
    2. Extract the new release archive
    3. Run ./setup.sh again — existing configuration is preserved
    4. Start the application: query-pulse start

    Truststore for Self-Signed Certificates¶

    If your data sources or authentication endpoints use self-signed certificates, configure a system truststore:

    export QUERY_PULSE_TRUSTSTORE_LOCATION=/path/to/truststore.jks
export QUERY_PULSE_TRUSTSTORE_PASSWORD=changeit

    This sets the JVM system properties so all outbound SSL connections (JDBC, LDAP, OIDC) trust the certificates in this truststore.

    You can also upload individual certificates through the UI under Settings > Certificates. See the Quick Start Guide for details on creating a truststore.

    Troubleshooting¶

    Issue Solution
    query-pulse: command not found Add $HOME/bin to your PATH or use the full path: $HOME/.query-pulse/bin/query-pulse.sh
    Java version error during setup Install JDK 17+ and set JAVA_HOME
    Port 8088 already in use Set QUERY_PULSE_SERVER_PORT to a different port
    Application starts but UI is blank Wait 30-60 seconds for full startup; check logs in $HOME/.query-pulse/logs/
    Cannot connect to data source Ensure the JDBC driver is installed and the truststore contains required certificates
