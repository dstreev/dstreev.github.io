    Driver Management¶

    Query Pulse provides comprehensive JDBC driver management, automatically downloading and configuring drivers for different database platforms and versions.

    Why Driver Management Matters¶

    Connecting to Hive, Impala, or Trino requires the correct JDBC driver for your specific platform and version. Mismatched drivers cause: - Connection failures - Authentication errors - Missing features - Compatibility issues

    Query Pulse eliminates driver complexity by: - Providing pre-configured driver definitions for all major platforms - Downloading drivers automatically from Maven repositories - Managing multiple driver versions simultaneously - Isolating drivers to prevent JAR conflicts

    Trino Drivers¶

    Trino drivers support version-aware management: - Default version: 446 - Any published version can be downloaded - Multiple versions can be installed simultaneously

    Drivers are stored locally for offline access:

    Installing Drivers¶

    Access Driver Management¶

    Admin Access Required: Only administrators can manage drivers.

    1. Click the Settings gear icon
    2. Select "JDBC Drivers" from Admin Settings

    Download a Driver¶

    1. Select the dialect tab (Hive or Trino)
    2. Find the driver for your platform
    3. Click "Install"
    4. Wait for download to complete
    5. Green checkmark indicates successful installation

    Driver Dependencies¶

    Each Hive driver downloads multiple JARs: - JDBC Driver: Main Hive JDBC standalone JAR - Hadoop Common: Core Hadoop classes - Hadoop Auth: Kerberos authentication support - Log4j 2: Logging dependencies

    Trino drivers download a single JAR containing all dependencies.

    Refresh a Driver¶

    To re-download a driver (e.g., after corruption):

    1. Find the installed driver
    2. Click "Refresh"
    3. Existing files are deleted and re-downloaded

    Maven Repository Integration¶

    Drivers are downloaded from: - Maven Central: Standard open-source JARs - Cloudera Repository: CDP/CDH-specific JARs

    Download progress is displayed during installation.

    Trino Version Management¶

    Available Versions¶

    Query Pulse fetches the list of available Trino versions from Maven Central metadata.

    Installing a Specific Version¶

    1. Select the Trino tab
    2. Choose a version from the dropdown
    3. Click "Install"

    Multiple Versions¶

    Install multiple Trino versions to support different clusters: - Each version is stored in a separate directory - Connections can specify which version to use - Class loaders are isolated per version

    Platform to Driver Mapping¶

    When creating connections, select your platform and Query Pulse automatically uses the correct driver:

    Platform Driver Used

    Apache Hive 3 Apache Hive 3
    Apache Hive 4 Apache Hive 4
    CDH 5 CDH 5 Hive
    CDH 6 CDH 6 Hive
    HDP 2 HDP 2 Hive
    HDP 3 CDP 7.1.9 Hive
    CDP 7.0-7.1.9 CDP 7.1.9 Hive
    CDP 7.2-7.3 CDP 7.3.1 Hive
    EMR 6/7 Apache Hive 3
    Trino Trino (selected version)

    Class Loader Isolation¶

    Query Pulse uses isolated class loaders for each driver:

    Benefits: - Prevent JAR conflicts between drivers - Support multiple driver versions simultaneously - Isolate driver dependencies from application code

    How It Works: 1. Each driver type gets its own URLClassLoader 2. Driver JARs are loaded in isolation 3. Connections use the appropriate class loader 4. Class loaders are cached for performance

    Driver Status¶

    Check driver installation status:

    Status Indicator Meaning
    Installed Green checkmark Driver ready to use
    Not Installed No indicator Driver needs download
    Installing Progress spinner Download in progress
    Error Red X Installation failed

    Troubleshooting¶

    Driver Not Found¶

    If connections fail with "driver not found": 1. Open Driver Management 2. Verify the correct driver is installed 3. Click Refresh to re-download if needed

    Version Mismatch¶

    If you see compatibility errors: 1. Check your cluster's actual Hive/Trino version 2. Select the matching platform in connections 3. Ensure the correct driver is installed

    Download Failures¶

    If downloads fail: 1. Check internet connectivity 2. Verify firewall allows Maven repository access 3. Try refreshing the driver

    Class Loading Errors¶

    If class loading fails: 1. Refresh the driver to re-download JARs 2. Restart the application to clear class loader cache 3. Check for JAR corruption in the driver directory

    Best Practices¶

    1. Install Before Connecting: Download drivers before creating connections
    2. Match Versions: Use drivers matching your cluster version
    3. Keep Updated: Refresh drivers when upgrading clusters
    4. Trino Versions: Install the version matching your Trino cluster
    5. Check Status: Verify green checkmarks before troubleshooting connections

    Configuration¶

    The driver base directory can be configured:

    app.driver.base-path=${user.home}/.query-pulse/drivers

    Default Trino version:

    trino.jdbc.version=446
