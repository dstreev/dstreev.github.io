    Hive Features¶

    Query Pulse provides comprehensive support for Apache Hive with real-time execution feedback, multiple explain plan visualizations, and detailed execution analytics.

    Real-Time Query Progress¶

    When executing queries against Hive connections, Query Pulse provides Beeline-style real-time progress tracking with detailed stage information.

    Progress Display¶

    The progress display shows live updates during query execution:

    • Overall Progress: Percentage completion with visual progress bar
    • Current Stage: Description of the currently executing stage
    • Elapsed Time: Human-readable duration since execution started

    Stage-Based Tracking¶

    Hive queries using Tez execution engine display detailed stage information:

    Stage Type Description
    Map Map-side processing stages
    Reducer Reduce-side processing stages
    Vertex Tez vertex execution stages

    Each stage displays task counters:

    • Total Tasks: Number of tasks in the stage
    • Completed: Successfully finished tasks
    • Running: Currently executing tasks
    • Pending: Tasks waiting to execute
    • Failed: Tasks that encountered errors
    • Killed: Tasks terminated before completion

    Stage Status Indicators¶

    Stages are color-coded by status:

    Status Color Description
    PENDING Gray Stage not yet started
    INITIALIZING Blue Stage initializing
    RUNNING Blue Stage actively executing
    COMPLETED Green Stage finished successfully
    FAILED Red Stage encountered an error
    KILLED Orange Stage was terminated

    View Modes¶

    Toggle between two display modes in User Preferences:

    • Summary View: Single-line progress bar with percentage
    • Detailed View: Full stage breakdown with all task counters

    Explain Plan Features¶

    Query Pulse provides five different explain visualizations for Hive queries, each offering unique insights into query execution.

    Visual Explain (EXPLAIN FORMATTED)¶

    Generates a graphical representation of the query execution plan.

    How to Use: 1. Write or select a query in the SQL editor 2. Click the Visual Explain button (tree icon) 3. A DOT graph file is created and opened in a new editor tab

    Output: Graphical execution plan showing: - Operators and their relationships - Data flow between stages - Join strategies - Filter pushdowns

    Command Executed: EXPLAIN FORMATTED <query>

    Visual Cost (EXPLAIN CBO)¶

    Shows the Cost-Based Optimizer's analysis of the query plan.

    How to Use: 1. Write or select a query in the SQL editor 2. Click the Visual Cost button (chart icon) 3. A DOT graph file is created showing the optimized plan

    Output: Calcite-based optimization plan displaying: - Cost estimates for each operation - Join ordering decisions - Predicate pushdown optimizations - Statistics-based decisions

    Command Executed: EXPLAIN CBO <query>

    Explain Dependency¶

    Analyzes table and partition dependencies for a query.

    How to Use: 1. Write or select a query in the SQL editor 2. Click the Explain Dependency button (JSON icon) 3. Results display as formatted JSON in the results panel

    Output: JSON structure showing: - Input tables required by the query - Partition dependencies - Database and table names - Dependency types

    Command Executed: EXPLAIN DEPENDENCY <query>

    Use Cases: - Understanding data lineage - Identifying partition pruning - Validating table access requirements

    Explain Authorization¶

    Shows the permissions required to execute a query.

    How to Use: 1. Write or select a query in the SQL editor 2. Click the Explain Authorization button (shield icon) 3. Results display as text in the results panel

    Output: Authorization details including: - Required read privileges - Required write privileges - Owner information - Table-level permissions

    Command Executed: EXPLAIN AUTHORIZATION <query>

    Use Cases: - Security auditing - Permission troubleshooting - Access control validation

    Explain Vectorization¶

    Analyzes vectorized execution capabilities for the query.

    How to Use: 1. Write or select a query in the SQL editor 2. Click the Explain Vectorization button (parallel lines icon) 3. A popup appears with configuration options 4. Select options and click "Execute"

    Configuration Options:

    Option Description
    ONLY Mode Show only vectorization details (exclude other plan info)
    Detail Level SUMMARY, DETAIL, or VERBOSE

    Detail Levels:

    • SUMMARY: High-level vectorization status
    • DETAIL: Operator-level vectorization information
    • VERBOSE: Complete vectorization details including expressions

    Output: DOT graph showing: - Which operators are vectorized - Why certain operators cannot be vectorized - Expression vectorization status - Data type compatibility

    Command Executed: EXPLAIN FORMATTED VECTORIZATION [ONLY] [SUMMARY|DETAIL|VERBOSE] <query>

    Execution Details¶

    After query completion, detailed execution information is available in the Execution Details panel.

    Query Metadata¶

    Field Description
    Query ID Hive-assigned query identifier
    DAG ID Tez DAG identifier
    Application ID YARN application ID
    Execution Time Total query duration
    Status Final query status

    Execution Summary¶

    When hive.tez.exec.print.summary=true is configured, the execution summary displays:

    Query Execution Summary: - Total MapReduce CPU time - Total HDFS bytes read/written - Total records processed

    Task Execution Summary: - Per-vertex task counts - Duration statistics (min, max, mean) - Input/output record counts

    Counter Sections: - HIVE counters - FileSystem counters - Shuffle error counters - Tez counters

    Supported Query Types¶

    Explain features are available for the following statement types:

    • SELECT
    • INSERT
    • UPDATE
    • DELETE
    • MERGE

    DDL statements (CREATE, ALTER, DROP) are not supported for explain operations.

    Variable Substitution¶

    All explain operations support workspace and file variables:

    -- Using variables in queries
SELECT * FROM ${database}.${table} WHERE date = '${run_date}'

    Variables are substituted before the explain command is executed. If undefined variables are detected, a dialog prompts for values.

    File Output¶

    Graphical explain operations create files in the workspace:

    Operation File Pattern Extension
    Visual Explain {filename}_explain_{n} .dot
    Visual Cost {filename}_cbo_{n} .dot
    Vectorization {filename}_vectorization_{n} .dot

    Files are automatically opened in new editor tabs for viewing.

    Best Practices¶

    1. Use Visual Explain to understand the basic execution structure
    2. Use Visual Cost when investigating performance issues related to join ordering or statistics
    3. Use Explain Dependency before making table changes to understand impact
    4. Use Explain Authorization when troubleshooting permission errors
    5. Use Explain Vectorization when queries are slower than expected to check if vectorization is being used
