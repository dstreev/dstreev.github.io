    SQL Code Assist¶

    Query Pulse provides intelligent SQL autocompletion to help you write queries faster and with fewer errors. Code Assist understands your database schema and provides context-aware suggestions as you type.

    Getting Started¶

    Code Assist is enabled by default. Simply start typing in the SQL editor, and suggestions will appear automatically.

    Triggering Completions¶

    Completions appear automatically when you:

    • Type after SQL keywords like SELECT, FROM, WHERE
    • Type a table alias followed by a dot (e.g., e.)
    • Start typing any identifier

    You can also press Ctrl+Space (or Cmd+Space on Mac) to manually trigger completions.

    Accepting Completions¶

    By default, press Tab to accept the highlighted suggestion. You can change this to Enter in your preferences.

    To navigate suggestions:

    • Up/Down arrows: Move through the list
    • Tab or Enter: Accept the selected suggestion
    • Escape: Dismiss the suggestions

    Completion Types¶

    Code Assist provides several types of suggestions:

    Icon Type Description
    keyword Keywords SQL keywords like SELECT, FROM, WHERE, JOIN
    function Functions Built-in SQL functions like CONCAT, SUM, AVG
    class Tables Table names from your database
    property Columns Column names from tables
    namespace Databases Database/schema names

    Context-Aware Suggestions¶

    Code Assist understands the context of your query and provides relevant suggestions.

    After SELECT¶

    When typing after SELECT, you'll see:

    • Column names from tables in your query
    • SQL functions
    • Keywords like DISTINCT, AS
    SELECT | -- Shows columns from tables in FROM clause, functions, keywords
FROM employees e
JOIN departments d ON e.dept_id = d.id

    After FROM or JOIN¶

    When typing after FROM or JOIN, you'll see table names from your database:

    SELECT *
FROM | -- Shows available tables

    After Table Alias with Dot¶

    When you type a table alias followed by a dot, you'll see only columns from that specific table:

    SELECT e.| -- Shows only columns from 'employees' table
FROM employees e
JOIN departments d ON e.dept_id = d.id

    Smart Column Filtering¶

    Code Assist automatically filters column suggestions based on the tables in your query:

    Without FROM clause: 

    SELECT | -- Shows all columns from all tables

    With FROM clause: 

    SELECT |
FROM employees e
JOIN departments d ON e.dept_id = d.id
-- Shows only columns from employees and departments

    This makes it easier to find relevant columns, especially in databases with many tables.

    Dialect Support¶

    Code Assist supports both Hive and Trino SQL dialects. The dialect is automatically detected based on your connection type.

    Hive-Specific Features¶

    • Hive keywords: LATERAL VIEW, EXPLODE, POSEXPLODE
    • Partitioning keywords: PARTITIONED BY, CLUSTERED BY
    • Hive functions: String, aggregate, and collection functions

    Trino-Specific Features¶

    • Trino keywords: CATALOG, MERGE, PREPARE, EXECUTE
    • Extended SQL syntax: FETCH FIRST, ROWS ONLY
    • Trino functions: Extended SQL standard functions

    Schema Loading¶

    For table and column suggestions to work, Code Assist needs to load your database schema.

    Schema Status Indicator¶

    The SQL editor shows the schema loading status:

    Status Meaning
    Loading... Schema is being loaded from the database
    Ready Schema loaded successfully
    Warning No database selected (run USE database_name)
    Disabled Schema assist disabled by administrator

    Refreshing Schema¶

    The schema cache is automatically refreshed periodically (default: every 24 hours). If you've made changes to your database schema, the updated schema will be available after the next refresh.

    If you need immediate updates, contact your administrator to manually refresh the cache.

    User Preferences¶

    You can customize Code Assist behavior in your preferences.

    Accessing Preferences¶

    1. Click your username in the top-right corner
    2. Select Preferences from the dropdown
    3. Scroll to the Code Assist section

    Available Settings¶

    Setting Options Description
    SQL Code Assist On/Off Enable or disable all SQL suggestions
    Schema Code Assist On/Off Enable or disable table/column suggestions
    Completion Accept Key Tab/Enter Key to accept suggestions

    Recommendations¶

    • Keep SQL Code Assist enabled for keywords and functions
    • Keep Schema Code Assist enabled if you work with unfamiliar databases
    • Use Tab for accept if you frequently use Enter to execute queries

    Troubleshooting¶

    No Suggestions Appearing¶

    1. Check your preferences: Ensure SQL Code Assist is enabled
    2. Check your connection: You need an active session for schema suggestions
    3. Select a database: Run USE database_name if you see a warning
    4. Wait for loading: Schema may still be loading for large databases

    Missing Tables or Columns¶

    1. Schema may be cached: Wait for the automatic refresh or contact your administrator
    2. Check permissions: You may not have access to certain tables
    3. Check the database: Ensure you're connected to the correct database

    Slow Suggestions¶

    1. Large schemas take longer: First load may be slow for databases with many tables
    2. Network latency: Suggestions require a round-trip to the server
    3. Subsequent requests are faster: Schema is cached after first load

    Wrong Dialect¶

    If you see keywords for the wrong SQL dialect:

    1. Check your connection configuration
    2. Ensure the connection type (Hive/Trino) is correctly set
    3. Reconnect if you changed the connection settings

    Tips and Best Practices¶

    Use Table Aliases¶

    Using short aliases makes it faster to get column suggestions:

    -- Good: Use short aliases
SELECT e.name, d.dept_name
FROM employees e
JOIN departments d ON e.dept_id = d.id

-- Less efficient: Typing full table names
SELECT employees.name, departments.dept_name
FROM employees
JOIN departments ON employees.dept_id = departments.id

    Start with FROM¶

    If you're exploring an unfamiliar database, start by typing the FROM clause:

    FROM | -- See available tables

    Then go back and add SELECT with context-aware column suggestions.

    Use Keywords for Navigation¶

    Type SQL keywords to narrow suggestions:

    SELECT * FROM employees WHERE | -- Shows columns for WHERE conditions
SELECT * FROM employees ORDER BY | -- Shows columns for ordering
SELECT * FROM employees GROUP BY | -- Shows columns for grouping

    Administrator Notes¶

    Administrators can configure Code Assist settings for all users:

    • Schema Cache TTL: How long schema metadata is cached (1 hour to 7 days)
    • Schema Assist Toggle: Enable or disable schema-aware completions globally

    See the Administration Guide for details on managing these settings.
