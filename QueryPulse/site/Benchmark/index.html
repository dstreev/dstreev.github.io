    Benchmarks¶

    Query Pulse includes industry-standard database benchmarks that can be added to your workspace for performance testing and analysis.

    What Are Benchmarks?¶

    Benchmarks are standardized sets of database schemas and queries developed by the Transaction Processing Performance Council (TPC). They provide:

    • Consistent Testing: Industry-accepted standards for comparing database performance
    • Real-World Scenarios: Queries that represent actual business workloads
    • Comprehensive Coverage: Wide range of query patterns and complexity levels

    Available Benchmarks¶

    TPC-DS (Decision Support Benchmark)¶

    The TPC-DS benchmark is designed for measuring decision support system performance with complex, real-world analytical queries.

    Attribute Value
    Tables 24
    Queries 99
    Focus Retail data warehouse analytics

    Schema Overview:

    TPC-DS models a retail product supplier with three sales channels:

    • Store Channel: Physical retail stores for in-person purchases
    • Catalog Channel: Mail-order catalog with phone/call center ordering
    • Web Channel: E-commerce website for online purchases

    The schema includes:

    • 7 Fact Tables: inventory, store_sales, store_returns, catalog_sales, catalog_returns, web_sales, web_returns
    • 17 Dimension Tables: date_dim, time_dim, customer, customer_address, customer_demographics, household_demographics, item, promotion, store, call_center, catalog_page, web_site, web_page, warehouse, ship_mode, reason, income_band

    Schema Files:

    File Description
    create_all_tables_w_constraints.sql Standard schema DDL with primary/foreign key constraints
    create_all_tables_w_constraints_documented.sql Fully documented schema with comprehensive column-level comments
    add_comments.sql ALTER TABLE statements to add comments to existing tables (for use with pre-existing TPC-DS databases)

    Choosing the Right Schema File:

    • New databases: Use create_all_tables_w_constraints_documented.sql to create tables with inline comments
    • Existing databases: Use add_comments.sql to add comments to tables that already exist (uses ALTER TABLE with the ${DB} variable)

    The add_comments.sql script is useful when you already have TPC-DS data loaded in databases like tpcds_text_100 or tpcds_bin_partitioned_orc_100 and want to enhance them with documentation:

    # Example: Add comments to existing TPC-DS database
hive --hivevar DB=tpcds_text_100 -f add_comments.sql

    Documented Schema for AI-Assisted SQL:

    The documented schema file includes rich metadata designed to enhance AI/LLM-assisted SQL generation:

    • Table-level comments: Business purpose, grain (what each row represents), and typical analysis use cases
    • Column-level comments: Semantic meaning, value formats, calculation formulas, and foreign key relationships
    • Business context: Explanations like "may differ for gifts" on shipping vs billing customer fields

    Example of documentation enhancement:

    -- Standard schema:
d_same_day_ly  INT,

-- Documented schema:
d_same_day_ly  INT  COMMENT 'Surrogate key of the same day last year, enables year-over-year comparisons',

    Benefits for Agentic SQL Creation:

    Benefit Description
    Semantic Understanding AI can understand business context without extensive prompting
    Accurate Joins Foreign key comments explain relationships, enabling correct join identification
    Query Explanations Rich metadata supports meaningful explanations to end users
    Reduced Errors Clear column semantics prevent misuse (e.g., using surrogate key vs natural key)

    This demonstrates how well-documented schemas can significantly improve the accuracy and usefulness of AI-generated SQL queries in analytical workflows.

    Query Categories:

    Category Queries Description
    Reporting 19 Standard business reports and operational queries
    Ad-hoc 36 One-time analytical and exploratory queries
    Iterative 32 Drill-down, roll-up, and slice-and-dice patterns
    Data Mining 12 Complex analytical and pattern discovery queries

    Adding Benchmarks to Your Workspace¶

    Method 1: Workspace Settings Panel¶

    1. Open your workspace
    2. Click the Settings icon in the workspace toolbar
    3. Select the Benchmarks tab
    4. Choose the benchmark you want to add (TPC-DS)
    5. Configure options:
    6. Database Name: Enter the target database name (optional)
    7. Include Schema: Check to add table creation scripts
    8. Include Queries: Check to add benchmark queries
    9. Click Setup Benchmark
    10. Refresh the file browser to see the new files

    Method 2: Benchmark Setup Modal¶

    1. Open your workspace
    2. Click the Add Benchmark button in the workspace panel
    3. Select the benchmark type
    4. Configure the same options as above
    5. Click Add to Workspace

    Workspace Structure After Setup¶

    When you add a benchmark, Query Pulse creates an organized directory structure:

    workspace/
└── tpcds/
    ├── schema/
    │   ├── create_all_tables_w_constraints.sql
    │   ├── create_all_tables_w_constraints_documented.sql
    │   └── add_comments.sql
    └── queries/
        ├── reporting/
        │   ├── query01.sql
        │   ├── query02.sql
        │   └── ...
        ├── ad_hoc/
        │   ├── query03.sql
        │   └── ...
        ├── iterative/
        │   ├── query04.sql
        │   └── ...
        └── data_mining/
            ├── query05.sql
            └── ...

    Database Variable¶

    When you specify a database name during setup, Query Pulse:

    1. Creates a workspace variable named DATABASE
    2. Sets its value to your specified database name
    3. All benchmark SQL files use ${DATABASE} placeholder

    This allows you to: - Run queries against different databases by changing one variable - Avoid hardcoding database names in SQL files - Easily switch between development and production environments

    Example usage in queries: 

    SELECT * FROM ${DATABASE}.store_sales
WHERE ss_sold_date_sk = 2451545;

    Running Benchmark Queries¶

    After adding a benchmark to your workspace:

    1. Create the Schema (if needed):
    2. Open tpcds/schema/create_all_tables_w_constraints.sql
    3. Ensure the DATABASE variable is set correctly

    4. Execute the script to create all tables

    5. Load Data:

    6. Use your preferred method to load TPC-DS data into the tables

    7. Data generators are available from the TPC website

    8. Run Queries:

    9. Open individual query files from the queries/ directories
    10. Execute queries to test performance
    11. Use the real-time progress tracking to monitor execution

    Setup Results¶

    After benchmark setup, you'll see a confirmation showing:

    • Number of schema files created
    • Number of query files created
    • List of created file paths
    • Variable added (if database name was specified)

    Best Practices¶

    1. Start with Schema: Always create the schema before running queries
    2. Use Variables: Leverage the ${DATABASE} variable for flexibility
    3. Category Selection: Start with simpler reporting queries before complex data mining queries
    4. Monitor Progress: Use Query Pulse's real-time feedback to track long-running benchmark queries
    5. Compare Results: Run benchmarks on different configurations to compare performance

    Use Cases¶

    • Performance Baseline: Establish baseline performance metrics for your cluster
    • Configuration Tuning: Test different Hive/Trino configurations
    • Hardware Comparison: Compare performance across different hardware
    • Regression Testing: Verify performance after upgrades or changes
    • Learning: Explore complex SQL patterns used in real-world analytics
