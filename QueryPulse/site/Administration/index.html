    System Administration¶

    This section covers database management, JVM configuration, LLM settings, and system monitoring.

    RocksDB Management¶

    Scope: Global (Admin Only)

    Manage the embedded RocksDB database used for all application data storage.

    Overview Tab¶

    Health Status: - Database open/closed status - Read/write capability verification

    Storage Statistics: - Total entries count - SST file count and size - Log file count and size

    Column Families:

    Family Purpose
    users User accounts
    workspaces Query workspaces
    queries Saved queries
    connections Database connections
    kerberos_configs Kerberos configurations
    tls_configs TLS configurations
    certificates SSL certificates
    keytabs Encrypted keytab files
    git_credentials Git authentication
    settings Application settings
    backups_metadata Backup tracking

    Data Browser Tab¶

    Browse and inspect stored data:

    1. Select column family
    2. Optionally filter by key prefix
    3. Browse keys with pagination
    4. View values (JSON formatted if applicable)
    5. Copy values to clipboard
    6. Delete individual keys

    Maintenance Tab¶

    Database Operations:

    Operation Description
    Compact Merge SST files to reclaim space
    Flush Write memtables to disk

    Backup Management:

    • Manual Backup: Create immediate backup
    • Scheduled Backups: Configure automatic backups (cron expression)
    • Backup Retention: Set number of backups to keep
    • Restore: Restore from any backup (requires restart)

    Dangerous Operations:

    • Clear Column Family: Delete all data in a column family
    • Requires confirmation dialog
    • Cannot be undone

    JVM Settings¶

    Scope: Global (Admin Only)

    Configure Java Virtual Machine parameters for optimal performance.

    Memory Settings¶

    Setting Description Default
    Initial Heap (-Xms) Starting heap size 256 MB
    Maximum Heap (-Xmx) Maximum heap size 1024 MB

    Garbage Collection¶

    GC Type Description Recommendation
    G1GC Balanced performance Recommended for most workloads
    ParallelGC Throughput optimized High-throughput batch processing
    SerialGC Single-threaded Small heaps, single-core
    ZGC Ultra-low latency Large heaps, latency-sensitive

    Network Settings¶

    • IPv4 Stack: Prefer IPv4 over IPv6

    Logging Configuration¶

    Setting Options
    Log Level TRACE, DEBUG, INFO, WARN, ERROR
    Log Prefix Custom log file prefix
    Log Directory Custom log file location

    LLM Memory Integration¶

    When LLM features are enabled: - Include LLM memory in heap calculations - Configurable LLM memory allocation - Helps prevent out-of-memory errors

    Additional JVM Arguments¶

    Free-form text field for advanced JVM options: 

    -XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/var/log/query-pulse/
-Dfile.encoding=UTF-8

    Status Display¶

    Current runtime information: - Max/Used/Free memory - Available processors - Java version and vendor - Configuration file path

    Note: Changes require application restart to take effect.

    LLM Configuration¶

    Scope: Global (Admin Only)

    Configure Large Language Model providers for AI-assisted query features.

    Provider Categories¶

    1. Embedded (Managed by Query Pulse)

    Ollama server managed by the application: - Download and install Ollama binary - Start/stop embedded server - Configure port (default: 11435) - Download and manage models

    Available models include: - llama3.2:3b (default) - codellama:7b - mistral:7b - And more from Ollama library

    2. Local (External LLM Servers)

    Connect to locally-running LLM servers:

    Provider Default Port
    Ollama 11434
    LM Studio 1234
    LocalAI 8080
    Custom User-defined

    3. Hosted (Cloud Providers)

    Connect to cloud-based LLM APIs:

    Provider Requirements
    OpenAI API Key
    Anthropic API Key
    Google AI API Key
    AWS Bedrock Access Key + Secret
    Azure OpenAI API Key + Endpoint
    Mistral API Key
    Groq API Key

    General Settings¶

    Setting Description Default
    Enable LLM Turn AI features on/off Off
    Max Output Tokens Response length limit 2048
    Context Length Input context size 4096
    Temperature Response randomness (0-1) 0.7

    Model Management¶

    For embedded Ollama: 1. View installed models 2. Browse available models 3. Download/pull new models 4. Delete unused models 5. Set active model

    System Status¶

    Scope: Global (All Users)

    View application health and status information.

    Available Information¶

    Application Status: - Version information - Uptime - Build timestamp

    Database Status: - RocksDB health - Storage statistics - Backup status

    Driver Status: - Installed drivers count - Available drivers count - Driver types summary

    LLM Status: - Provider configuration - Model availability - Server status (if embedded)

    Memory Status: - JVM heap usage - Available memory - GC statistics
