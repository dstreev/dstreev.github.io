    Cloudera Manager Installation¶

    Deploy Query Pulse as a managed service on a CDP cluster using a CSD (Custom Service Descriptor) and Parcel.

    Prerequisites¶

    • Cloudera Manager 7.0 or later
    • CDP cluster with at least one host available for the Query Pulse role
    • Admin access to Cloudera Manager

    Step 1 — Install the CSD¶

    The CSD (Custom Service Descriptor) tells Cloudera Manager how to configure and manage the Query Pulse service. It must be installed before the parcel.

    Obtain the CSD JAR (QUERY_PULSE-<version>.jar) from the release artifacts.

    1. Copy the CSD JAR to /opt/cloudera/csd/ on the Cloudera Manager host
    2. Restart the Cloudera Manager Server: 
      sudo systemctl restart cloudera-scm-server
    3. After restart, Query Pulse will appear as an available service type

    Step 2 — Install and Activate the Parcel¶

    The parcel contains the Query Pulse application binaries that get distributed to cluster hosts.

    Obtain the parcel (QUERY_PULSE-<version>-<distro>.parcel) and its SHA checksum file from the release artifacts.

    1. Copy the .parcel and .parcel.sha files to /opt/cloudera/parcel-repo/ on the Cloudera Manager host
    2. In Cloudera Manager, go to Parcels and click Check for New Parcels
    3. Find QUERY_PULSE in the list
    4. Click Distribute and wait for distribution to all hosts to complete
    5. Click Activate to make the parcel available for use

    Step 3 — Add the Service¶

    1. In Cloudera Manager, go to your cluster and click Add Service
    2. Select Query Pulse from the service list
    3. Assign the Query Pulse Server role to a host
    4. Configure the service (see Configuration below)
    5. Click Start to launch the service

    Configuration¶

    All settings are available in the Cloudera Manager UI under the Query Pulse service configuration.

    Core Settings¶

    Parameter Default Description
    Server Port 8088 HTTP/HTTPS port
    Context Path / URL context path
    Working Directory /var/lib/query-pulse Data and configuration directory
    Log Directory /var/log/query-pulse Application log files

    JVM Settings¶

    Parameter Default Description
    Heap Initial Size 2048 MB Initial JVM heap
    Heap Max Size 4096 MB Maximum JVM heap
    Garbage Collector G1GC GC algorithm (G1GC, ZGC, ParallelGC)
    Log Level INFO Application log verbosity

    Authentication¶

    Parameter Default Description
    Authentication Type local local, ldap, or oidc
    Admin Username admin Admin account username
    Reset Admin Password false Reset admin password on next start

    LDAP and OIDC parameters are also configurable through the CM UI. See the Quick Start Guide for parameter details.

    Auto-TLS¶

    When Auto-TLS is enabled on the cluster, Query Pulse automatically discovers and uses the cluster's keystores and truststores. The control script searches for:

    • cm-auto-host_keystore.jks — Server keystore
    • cm-auto-host_key.pw — Keystore password
    • cm-auto-in_cluster_truststore.jks — In-cluster truststore (preferred)
    • cm-auto-global_truststore.jks — Global truststore (fallback)

    No manual truststore configuration is needed when Auto-TLS is active.

    For custom certificates, configure the Client Truststore Location and Client Truststore Password in the SSL/TLS section of the service configuration.

    Web Links¶

    After the service starts, Cloudera Manager provides quick links:

    Link URL
    Query Pulse Web UI http://<host>:<port>/
    API Docs (Swagger) http://<host>:<port>/swagger-ui.html
    Health Status http://<host>:<port>/actuator/health

    Service Management¶

    Once deployed, manage Query Pulse through Cloudera Manager like any other service:

    • Start / Stop / Restart from the service Actions menu
    • View logs from the Query Pulse Server role Logs tab
    • Monitor health via the built-in health check
    • Update configuration and rolling restart when needed

    Upgrading¶

    To upgrade to a new version:

    1. Obtain the new parcel and CSD JAR from the release artifacts
    2. Replace the CSD JAR in /opt/cloudera/csd/ and restart CM Server
    3. Upload the new parcel to /opt/cloudera/parcel-repo/
    4. In Cloudera Manager: Distribute, Activate the new parcel, Deactivate the old one
    5. Restart the Query Pulse service

    Troubleshooting¶

    Issue Solution
    Service fails to start Check stderr log in CM role instance Logs tab
    Parcel not visible in CM Verify .parcel and .sha files are in /opt/cloudera/parcel-repo/ and click Check for New Parcels
    CSD not visible after install Restart the Cloudera Manager Server process
    SSL handshake failures Verify Auto-TLS is active or configure a custom truststore
    Java version error Ensure the host has JDK 17+ and the parcel's defines.sh points to it
