    Real-Time Feedback¶

    Query Pulse provides real-time execution feedback for long-running queries, giving you visibility into query progress instead of waiting blindly for completion.

    Why Real-Time Feedback Matters¶

    Traditional SQL tools show a spinning cursor while queries execute. You have no idea: - Is the query still running or stuck? - How much longer will it take? - Which stage is currently executing? - Are there any failures in progress?

    Query Pulse solves this with streaming progress updates that show exactly what's happening during query execution.

    How It Works¶

    Query Pulse uses Server-Sent Events (SSE) to stream progress updates from the database engine to your browser in real-time:

    Browser ←──SSE Stream──← Backend ←──Progress API──← Database Engine

    Progress updates arrive as events containing: - Current execution stage - Task completion counts - Elapsed time - Error information (if any)

    Engine-Specific Progress¶

    Hive Progress Tracking¶

    Hive queries using the Tez execution engine provide detailed Beeline-style progress:

    Stage Information: - Map stages processing input data - Reducer stages aggregating results - Vertex stages in the execution DAG

    Task Counters per Stage:

    Counter Description
    Total Number of tasks created
    Completed Tasks finished successfully
    Running Tasks currently executing
    Pending Tasks waiting to start
    Failed Tasks that encountered errors
    Killed Tasks terminated early

    Stage Status Colors:

    Status Color Meaning
    PENDING Gray Not started
    INITIALIZING Blue Setting up
    RUNNING Blue Actively executing
    COMPLETED Green Finished successfully
    FAILED Red Error occurred
    KILLED Orange Terminated

    Execution Summary:

    When hive.tez.exec.print.summary=true is configured, additional details are available after completion: - Query Execution Summary with CPU time and I/O statistics - Task Execution Summary with per-vertex metrics - Counter breakdowns (HIVE, FileSystem, Shuffle)

    Trino Progress Tracking¶

    Trino provides comprehensive query statistics through its Query Stats API:

    Query State:

    State Description
    QUEUED Waiting in queue
    PLANNING Generating execution plan
    RUNNING Actively executing
    FINISHING Completing final operations
    FINISHED Completed successfully
    FAILED Error occurred

    Task Metrics: - Total, Running, Completed, Failed task counts

    Driver Metrics: - Total, Running, Queued, Blocked, Completed driver counts

    Data Processing: - Physical input data size and row counts - Processed input data size and row counts

    Timing Breakdown:

    Metric Description
    Elapsed Time Total wall-clock time
    Execution Time Time spent executing
    Queued Time Time waiting in queue
    Analysis Time Query analysis duration
    Planning Time Distributed planning duration
    CPU Time Total CPU consumed
    Scheduled Time Total scheduled time
    Blocked Time Time waiting for resources

    Memory Metrics: - Peak user memory reservation - Peak total memory reservation - Current memory reservation

    Progress Display¶

    Summary View¶

    A compact single-line display showing: - Progress bar with percentage - Current stage description - Elapsed time

    Best for: Quick status checks, limited screen space

    Detailed View¶

    Expanded display showing: - All stage information with task counters - Complete timing breakdown - Memory and resource metrics

    Best for: Debugging, performance analysis, detailed monitoring

    Switching Views¶

    Set your preferred view in User Preferences: 1. Open Settings dropdown 2. Select Preferences 3. Choose Query Progress View: Summary or Detailed

    Execution Details Panel¶

    After query completion, the Execution Details panel provides comprehensive information:

    Hive Details¶

    • Query ID: Hive-assigned identifier
    • DAG ID: Tez DAG identifier
    • Application ID: YARN application ID
    • Execution Time: Total duration
    • Status: Final query status
    • Execution Summary: Parsed summary tables and counters

    Trino Details¶

    • Query Statistics Grid: Organized metrics by category
    • Operator Summaries: Per-operator execution statistics
    • Expandable Details: Click operators for detailed metrics
    • Spill Detection: Highlighted warnings for disk spills

    Canceling Queries¶

    Long-running queries can be canceled:

    1. Click the "Cancel" button during execution
    2. The query is terminated on the server
    3. Progress display shows cancellation status

    Benefits¶

    Benefit Description
    Visibility Know exactly what's happening during execution
    Early Detection Spot problems before queries complete
    Performance Insight Identify slow stages and bottlenecks
    Time Estimation Progress percentage helps estimate completion
    Resource Monitoring Track memory and CPU usage
    Debugging Detailed metrics for troubleshooting

    Supported Operations¶

    Real-time feedback is available for:

    • Single statement execution
    • Script execution (per-statement progress)
    • Explain operations
    • Validation operations

    Best Practices¶

    1. Monitor Failed Tasks: High failure counts may indicate data or resource issues
    2. Watch Blocked Time: High blocked time suggests resource contention
    3. Track Memory: Approaching peak memory may cause spills
    4. Check Stage Balance: Uneven task counts may indicate data skew
    5. Use Detailed View: When debugging performance issues
    6. Use Summary View: For routine query monitoring
