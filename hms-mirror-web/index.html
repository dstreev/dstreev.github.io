Data Strategies

Pick the Right Migration Path

Each strategy handles location translation, table type conversion, partition management, and view migration independently. Mix and match across jobs.

  • SCHEMA_ONLY — Transfer DDL without data movement; external tools handle the copy
  • SQL — Data movement via INSERT...SELECT using shadow tables as bridges
  • HYBRID — Auto-selects strategy based on table size and complexity
  • STORAGE_MIGRATION — Move data to new HDFS/cloud namespaces within a cluster
  • EXPORT_IMPORT — Native Hive export/import for robust cross-cluster transfer

Table Modernization

ACID Tables & Iceberg Conversion

Handle the hardest part of Hive migration: transactional tables. HMS-Mirror manages ACID version incompatibilities, format conversion, and in-place Iceberg upgrades.

  • ACID downgrade — Convert transactional tables to external with EXTERNAL_TABLE_PURGE
  • Bucket removal — Automatic artificial bucket translation from Hive 2 to Hive 3
  • Iceberg v1 & v2 — In-place table format conversion without data movement
  • Format translation — ORC, Parquet, and Avro format handling during migration

Storage

Intelligent Location Translation

Automatically remap table and partition locations from source to target namespaces. Supports HDFS-to-cloud migrations with intelligent path resolution.

  • ALIGNED mode — Warehouse plan-based restructuring for modern data layouts
  • RELATIVE mode — Namespace swap while preserving directory structure
  • Custom mappings — User-defined path translations with longest-match semantics
  • Cloud targets — S3A, ADLS (abfs://), and GCS (gs://) native support

Deploy in 5 Minutes

Choose your deployment method. From a single JAR to Cloudera Manager integration.

1

Download the server JAR

Shell 
curl -L -o hmsm-server.jar \
  https://github.com/cloudera-labs/hms-mirror/releases/latest/download/hmsm-server.jar
2

Start the server

Shell 
java -jar hmsm-server.jar

Web UI available at http://localhost:8090/hms-mirror

3

Create your first connection and run a migration

Use the web UI wizard to configure source/target connections, scope your dataset, and execute.

Requirements: Java 17+ • 4 GB RAM minimum • Network access to source and target clusters
1

Pull and run the Docker image

Shell 
docker run -d --name hms-mirror \
  -p 8090:8090 \
  -v hms-data:/data/hms-mirror \
  -v hms-logs:/var/log/hms-mirror \
  cloudera/hmsm-server:latest

Rocky Linux 9 base with Java 17 and all dependencies included.

2

Open the web UI

Shell 
open http://localhost:8090/hms-mirror

Health check: /hms-mirror/actuator/health

Requirements: Docker 20+ • 8 GB RAM recommended • Ports 8090 available
1

Upload the CSD to Cloudera Manager

Shell 
scp HMS_MIRROR-*.jar admin@cm-host:/opt/cloudera/csd/
systemctl restart cloudera-scm-server
2

Distribute and activate the Parcel

Upload the .parcel and .parcel.sha files, then distribute and activate through CM UI. Supports EL7, EL8, EL9, and Ubuntu.

3

Add the HMS-Mirror service

Add Service → HMS-Mirror. Configure Kerberos principal, TLS, and heap settings through CM. Min CDH version: 7.0.0.

Requirements: Cloudera Manager 7.0+ • CDP cluster • Admin access to CM

Built for the Enterprise

Security, compliance, and operational features for production deployments.

PKCS12 Keystore Encryption

Master keys stored in a hardware-grade PKCS12 keystore. All sensitive configuration values encrypted at rest.

JWT Authentication

15-minute access tokens with 7-day refresh tokens. Configurable secrets with auto-generation fallback.

LDAP & OIDC SSO

Active Directory, IPA, and custom LDAP integration. OpenID Connect for browser-based SSO with Okta and Azure AD.

Role-Based Access

Admin and user roles with group mapping from LDAP/OIDC. Admin account always uses local auth as failsafe.

TLS/SSL Everywhere

Certificate management for cluster connections, StartTLS for LDAP, and configurable SSL for the web UI.

Cloudera Manager Native

CSD and Parcel deployment with CM health checks, external links, Kerberos principal support, and auto-configuration.
