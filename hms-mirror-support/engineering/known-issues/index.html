    Known Issues & Workarounds¶

    This page tracks known limitations, workarounds, and technical debt notes that engineering and L3 should be aware of.

    Active Issues¶

    KI-001: YAML Code Point Limit for Very Large Reports¶

    Symptom: Parsing error when loading a conversion result YAML that exceeds 100MB.

    Root Cause: SnakeYAML's default code point limit is 3MB. HMS-Mirror raises this to 100MB, but very large migrations (thousands of tables with many partitions) can exceed this.

    Workaround: Increase the limit: 

    hms-mirror.config.yaml.codePointLimit=209715200  # 200MB

    Engineering Note: Long-term fix is to switch large reports to streaming JSON rather than YAML.

    KI-002: RocksDB LOCK File Not Cleaned on Crash¶

    Symptom: After an ungraceful shutdown (kill -9, Docker restart, OOM kill), HMS-Mirror reports "Cannot acquire lock" on next startup.

    Root Cause: RocksDB uses an advisory lock file (~/.hms-mirror/data/LOCK) that isn't released on crash.

    Workaround: 

    rm -f ~/.hms-mirror/data/LOCK

    Engineering Note: The startup script should detect and handle stale lock files. Tracked for future cleanup.

    KI-003: Socket.IO Client Dependency Removed¶

    Context: The socket.io-client npm package was present in package.json but was never imported in the TypeScript codebase. Removed in v4.1.5.8 as part of T19 remediation.

    Impact: None — the package was unused.

    Engineering Note: If Socket.IO is needed in the future (for bidirectional real-time), it must be re-added with full implementation. The current real-time mechanism is SSE (Spring SseEmitter).

    KI-004: JWT Sessions Invalidated on Restart Without Persistent Secret¶

    Symptom: All users are logged out whenever the server restarts.

    Root Cause: If hms-mirror.jwt.secret is empty, a new random secret is generated at each startup. All existing tokens become invalid.

    Workaround: Set a persistent JWT secret in web.properties: 

    hms-mirror.jwt.secret=<random-64-char-value>

    Generate with: 

    openssl rand -base64 64

    Engineering Note: This is documented behavior, not a bug. However, the documentation could be more prominent during first-run setup.

    KI-005: No Concurrent Job Execution Limit¶

    Symptom: Running too many jobs simultaneously can cause OOM or resource contention.

    Root Cause: No server-side limit on concurrent running jobs.

    Workaround: Don't run more than 2-3 jobs simultaneously. Schedule large jobs serially.

    Engineering Note: A job queue with configurable concurrency is planned.

    KI-006: Views Not Migrated by Default¶

    Symptom: Customer reports views are missing on target after migration.

    Root Cause: View migration is disabled by default in the ConfigLite template.

    Workaround: 

    migrate:
  views: true

    Engineering Note: This should be more visible in the UI — consider adding a warning when views are detected but migration is disabled.

    Design Decisions and Rationale¶

    Why RocksDB Instead of a Traditional RDBMS?¶

    RocksDB was chosen as the embedded database for: - Zero external service dependency (single JAR deployment) - High write throughput for large migration result storage - Schema flexibility for evolving domain models

    Trade-off: No SQL query capability, limited tooling for direct inspection.

    Why PKCS12 for the Encryption Keystore?¶

    PKCS12 is the standard Java keystore format as of Java 9+. The older JKS format is deprecated. PKCS12 is widely supported by keytool and openssl.

    Why JWT Instead of Sessions?¶

    JWT enables stateless API authentication, which supports: - Multiple server instances (no shared session store needed) - REST API usage without browser cookies - CLI and programmatic access patterns

    Trade-off: Token revocation requires rotating the secret (affects all users). Per-token blacklist is a known enhancement.

    Technical Debt Items¶

    Item Priority Notes
    Job execution queue Medium Prevent unbounded concurrent jobs
    Per-token revocation Low JWT blacklist for individual session invalidation
    Streaming JSON for large reports Low Alternative to YAML for very large ConversionResult
    Startup lock file handling Low Auto-detect and warn about stale LOCK files
    Admin audit log High Track all admin actions to a dedicated audit log
