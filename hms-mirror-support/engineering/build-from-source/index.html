    Build from Source¶

    Prerequisites¶

    Requirement Version Notes
    Java JDK 17+ Must be JDK, not just JRE
    Maven 3.8+ mvn --version
    Node.js 20.18.0 Managed by frontend-maven-plugin; manual install not required
    npm Bundled with Node Managed automatically
    Git Any recent For source checkout
    Internet access Required for first build Maven downloads dependencies

    Project Structure¶

    hms-mirror/
├── pom.xml                    ← Reactor parent POM
├── common/                    ← Shared domain models (no Spring)
├── server/                    ← Spring Boot application
│   ├── src/main/frontend/     ← React 19 + Vite frontend
│   └── bin/hmsm-server        ← Startup script
├── cli/                       ← CLI wrapper
├── client/                    ← REST client library
├── test-fixtures/             ← Test YAML data (no Java)
├── csd/HMS_MIRROR/            ← CM Service Descriptor (standalone POM)
└── parcel/                    ← CM Parcel packaging (standalone POM)

    CSD and Parcel are Standalone

    The csd/ and parcel/ modules have their own POMs and are not part of the Maven reactor. Build them separately. The update_version.sh script handles their versioning.

    Build Commands¶

    Full Build (All Modules + Frontend)¶

    mvn clean package -DskipTests

    Output: server/target/hmsm-server-<version>-server.jar

    Server Only (Faster Iteration)¶

    mvn clean package -pl server -am -DskipTests

    The -am flag also builds the common dependency.

    With Tests¶

    # All tests
mvn test

# Server tests only
mvn test -pl server

# Specific test class
mvn test -pl server -Dtest=AdminUserCreationServiceTest

# Specific test method
mvn test -pl server -Dtest=AdminUserCreationServiceTest#createUser_withAdminFlag_grantAdminRoleToNewUser

    CSD Build (Standalone)¶

    cd csd/HMS_MIRROR
mvn clean package
# Output: target/HMSMIRROR-<version>.jar

    Parcel Build (Standalone)¶

    cd parcel
mvn clean package -P el9   # el7, el8, el9, ubuntu
# Output: target/HMSMIRROR-<version>-el9.parcel + .sha

    Frontend Build¶

    The React frontend is built automatically during mvn package. To build it independently:

    cd server/src/main/frontend
npm install
npm run build         # Production build
npm run dev           # Dev server with hot reload (port 5173)

    The Vite dev server proxies API calls to http://localhost:8090. The production build is bundled into the server JAR.

    Version Management¶

    The version format is X.Y.Z.B (e.g., 4.1.5.8). Version is set in multiple places:

    # Preview changes
./update_version.sh 4.1.5.9 --dry-run

# Apply version change to all files
./update_version.sh 4.1.5.9

# Files updated:
# - pom.xml (all Maven modules)
# - server/src/main/frontend/package.json
# - Dockerfile
# - docs/
# - csd/HMS_MIRROR/pom.xml (standalone)
# - parcel/pom.xml (standalone)

    Running Locally¶

    # Build and run
mvn clean package -pl server -am -DskipTests
java -jar server/target/hmsm-server-*-server.jar

# With debug port
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005 \
  -jar server/target/hmsm-server-*-server.jar

# With custom working directory
java -jar server/target/hmsm-server-*-server.jar \
  --hms-mirror.working-directory-base=/tmp/hms-test

    UI at: http://localhost:8090/hms-mirror

    Running in Docker¶

    # Build image (after mvn package)
docker build -t hms-mirror:local .

# Run with persistent data
docker run -p 8090:8090 \
  -v ~/.hms-mirror:/root/.hms-mirror \
  hms-mirror:local

    Test Infrastructure¶

    E2E Test Pattern¶

    E2E tests extend ServerE2EBaseTest which provides RocksDB isolation per test:

    @Test
void testSchemaOnly() throws Exception {
    // Load test fixtures
    ConnectionDto source = loadConnection("test-fixtures/config/source.yaml");
    ConnectionDto target = loadConnection("test-fixtures/config/target.yaml");
    DatasetDto dataset = loadDataset("test-fixtures/datasets/default.yaml");

    // Build ConversionResult
    ConversionResult result = buildConversionResult(source, target, dataset, Strategy.SCHEMA_ONLY);

    // Run the engine
    appService.run(result);

    // Validate results
    E2EValidationHelper.assertTableSuccess(result, "sales_db", "orders");
}

    Test Fixtures Location¶

    test-fixtures/src/main/resources/
├── config/          ← Connection/ConfigLite YAMLs
├── datasets/        ← Dataset YAML files
├── test_data/       ← Table definition fixtures
└── expected_results/← Per-scenario expected output
    ├── cdp_to_cdp/
    ├── schema_only_hdp2_to_cdp/
    └── ...
