    Installation Verification¶

    Use this page when the customer says the server won't start, won't respond, or is returning errors at the URL.

    Step 1: Confirm the Server Process Is Running¶

    # If installed via service script
/path/to/hmsm-server status

# Or check the PID file
cat ~/.hms-mirror/hms-mirror.pid

# Or check the process directly
ps aux | grep hmsm-server

    If the PID file exists, confirm the process is actually running:

    PID=$(cat ~/.hms-mirror/hms-mirror.pid)
kill -0 $PID 2>/dev/null && echo "RUNNING" || echo "DEAD (stale PID file)"

    Step 2: Test the Health Endpoint¶

    curl -s http://localhost:8090/hms-mirror/api/v1/system/health

    Expected responses:

    Response Meaning
    {"status":"UP"} Server is healthy
    {"status":"DEGRADED"} Running but with warnings (check logs)
    {"status":"DOWN"} Running but not operational
    Connection refused Server is not running on that port
    HTTP 404 Wrong URL — check context path

    Wrong Port?

    The default port is 8090. If the customer configured a different port, check ~/.hms-mirror/settings/web.properties for server.port.

    Step 3: Check the Startup Log¶

    The log captures the full startup sequence:

    tail -200 ~/.hms-mirror/logs/hms-mirror.log

    Look for these patterns:

    Pattern Meaning
    Started Mirror in X.X seconds Successfully started
    Port 8090 was already in use Port conflict — another process on that port
    Unable to open RocksDB Database locked or corrupted
    Keystore not found Security keystore missing — check ~/.hms-mirror/security/
    Address already in use Port in use by another process
    Could not find or load main class Corrupted JAR or wrong startup command

    Step 4: Verify Java Version¶

    HMS-Mirror requires Java 17 or higher:

    java -version

    If the version is below 17, the server will fail at startup with: 

    UnsupportedClassVersionError

    Step 5: Verify Working Directory Permissions¶

    ls -la ~/.hms-mirror/

    The process user must have read/write access to:

    Directory Purpose
    ~/.hms-mirror/data/ RocksDB embedded database
    ~/.hms-mirror/logs/ Application logs
    ~/.hms-mirror/security/ JWT keystore
    ~/.hms-mirror/settings/ Configuration overrides
    ~/.hms-mirror/reports/ Migration output reports

    Step 6: Check Port Availability¶

    # Check if port 8090 is already in use
lsof -i :8090
# or
ss -tlnp | grep 8090

    If another process is using port 8090, either stop that process or configure HMS-Mirror to use a different port in ~/.hms-mirror/settings/web.properties:

    server.port=8091

    Common Startup Failures¶

    "Address already in use: 8090"¶

    Another process owns port 8090. Options: 1. Stop the other process 2. Change HMS-Mirror's port in web.properties 3. Check if HMS-Mirror is already running (double-start)

    "Cannot open RocksDB — resource temporarily unavailable"¶

    Only one HMS-Mirror instance can access the RocksDB database at a time. Another instance is already running, or crashed with an open lock.

    # Find and remove stale lock
rm -f ~/.hms-mirror/data/LOCK

    Server starts but UI shows blank page¶

    1. Confirm you're using the correct URL: http://host:8090/hms-mirror (not just http://host:8090)
    2. Clear browser cache and hard refresh (Ctrl+Shift+R)
    3. Check browser console for JavaScript errors

    HTTPS required / Certificate errors¶

    If TLS is configured, the URL must be https://. Check ~/.hms-mirror/settings/tls.properties for TLS configuration.
