    L1 Frontline Support¶

    Audience: Customer-facing support agents handling first contact.

    Your job: Verify the product is running correctly, confirm the customer can log in, collect initial diagnostic information, and either resolve simple issues directly or escalate with a complete data package.

    What L1 Handles¶

    • Verify the server is running and accessible
    • Confirm login works (or identify why it doesn't)
    • Guide customers through first-run setup
    • Collect the standard support data package
    • Answer "what does this product do?" questions
    • Make the escalation decision

    What L1 Does NOT Handle¶

    • Log file interpretation beyond "is it running?"
    • JDBC or Kerberos connection debugging
    • Migration failure analysis
    • Configuration changes

    L1 Workflow¶

    Customer contacts support
        │
        ▼
1. Is the server running?  ──NO──▶  Installation Verification
        │ YES
        ▼
2. Can the user log in?  ──NO──▶  Login Issues
        │ YES
        ▼
3. What is the customer trying to do?
        │
        ├──▶ First-run setup question  ──▶  First-Run Setup
        │
        ├──▶ Migration failing  ──▶  Basic Triage → Escalate L2
        │
        └──▶ Other  ──▶  Basic Triage → Data Collection → Escalate
