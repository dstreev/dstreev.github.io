    Architecture Primer¶

    Understanding how HMS-Mirror is built helps you know where to look when something goes wrong.

    Key Subsystems¶

    RocksDB (Embedded Database)¶

    All persistent application data lives in RocksDB — an embedded key-value store at ~/.hms-mirror/data/.

    What's stored: - User accounts and roles - Connection definitions - Configuration (ConfigLite) definitions - Dataset definitions - Job definitions and run history - Migration results (ConversionResult)

    Why it matters for support: - If RocksDB is corrupted, the application cannot start - You cannot edit it directly with standard tools — changes go through the API - The only safe recovery is restore from backup or reset

    JWT Security Layer¶

    All API endpoints require authentication. The filter chain:

    HTTP Request
    │
    ▼
JwtAuthenticationFilter  ← reads Bearer header or ?token= query param
    │
    ▼
LoginRateLimitFilter     ← blocks brute-force on /auth/login
    │
    ▼
Spring Security          ← evaluates @PreAuthorize annotations
    │
    ▼
Controller Method

    Token lifetimes: - Access token: 15 minutes - Refresh token: 7 days

    Migration Engine¶

    When a Job runs, the engine:

    1. Resolves the Connection, Config, and Dataset references into a ConversionResult
    2. Creates SQL statements for each table based on the chosen strategy
    3. Executes those statements against source and/or target clusters via JDBC
    4. Records results (SUCCESS/FAILURE per table) in the ConversionResult
    5. Generates the output report

    Connection Layer¶

    HMS-Mirror connects to Hive via JDBC. Each connection in the application maps to one or two actual JDBC connections at runtime (source + target). JDBC drivers are loaded dynamically from ~/.hms-mirror/aux_libs/ and ~/.hms-mirror/drivers/.

    Data Flow: What Happens During a Migration¶

    1. User submits Job via UI/API
        │
2. Engine loads Connection + Config + Dataset
        │
3. Engine connects to SOURCE via JDBC
        ├── Lists databases matching Dataset scope
        └── For each database, lists tables
        │
4. For each table:
        ├── Reads table DDL from SOURCE metastore
        ├── Generates target DDL (adjusted for strategy)
        ├── Executes target DDL against TARGET
        └── (For data-moving strategies) runs INSERT/EXPORT/IMPORT
        │
5. Records per-table result (SUCCESS/FAILURE + SQL executed)
        │
6. Saves ConversionResult to RocksDB
        │
7. Generates reports in ~/.hms-mirror/reports/<job-id>/

    Configuration File Hierarchy¶

    Property resolution order (highest priority first):

    1. Environment variables (e.g., HMS_MIRROR_SERVER_PORT)
    2. ~/.hms-mirror/settings/web.properties
    3. ~/.hms-mirror/settings/tls.properties
    4. Built-in Spring Boot defaults

    Key config files:

    File Purpose
    ~/.hms-mirror/settings/web.properties Port, auth type, JWT settings, feature flags
    ~/.hms-mirror/settings/tls.properties TLS keystore path and password
    ~/.hms-mirror/logs/hms-mirror.log Application logs
    ~/.hms-mirror/data/ RocksDB database directory
    ~/.hms-mirror/security/ PKCS12 encryption keystore
    ~/.hms-mirror/aux_libs/ Additional JARs (JDBC drivers)
    ~/.hms-mirror/reports/ Migration output reports

    Spring Boot Context Path¶

    All HMS-Mirror URLs are under /hms-mirror:

    • Web UI: http://host:8090/hms-mirror
    • REST API: http://host:8090/hms-mirror/api/v1/
    • Documentation: http://host:8090/hms-mirror/docs/

    Requests to the root path / return 404. This is by design and not a bug.

    Module Layout (for Log Identification)¶

    Log entries include the Java package name. Knowing the modules helps identify the source:

    Log Package Prefix Subsystem
    c.c.u.h.m.web.controller REST API controllers
    c.c.u.h.m.service Business logic services
    c.c.u.h.m.security Authentication/JWT
    c.c.u.h.m.repository RocksDB data access
    c.c.u.h.m.datastrategy Migration strategy engine
    c.c.u.h.m.config Spring configuration
    o.s.security Spring Security framework
    o.a.hive Hive JDBC driver
