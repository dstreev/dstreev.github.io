    Configuration Issues¶

    Configuration errors account for a significant portion of "it doesn't work" cases. This page covers how to diagnose and fix them.

    Configuration Files Overview¶

    File Purpose When it matters
    ~/.hms-mirror/settings/web.properties Server settings: port, auth, JWT, features Server won't start, wrong port, auth failures
    ~/.hms-mirror/settings/tls.properties TLS/HTTPS settings Certificate errors, HTTPS required
    Connection config (stored in RocksDB) JDBC URIs, credentials Connection test failures
    ConfigLite (stored in RocksDB) Migration settings Unexpected migration behavior
    Dataset (stored in RocksDB) Table scope Missing or extra tables in migration

    web.properties Common Mistakes¶

    View the current settings: 

    cat ~/.hms-mirror/settings/web.properties

    Wrong Port¶

    # Customer set this but forgot to update firewall
server.port=9090

    Result: UI accessible at port 9090, not 8090. Health check at localhost:8090 fails.

    JWT Secret Too Short¶

    hms-mirror.jwt.secret=short

    The JWT secret should be at least 32 characters. If it's blank, a random secret is generated at each startup — this means all sessions are invalidated on restart.

    # Correct: persistent 64-char secret
hms-mirror.jwt.secret=your-long-random-secret-at-least-32-chars-here-please-use-more

    Wrong Authentication Type¶

    # Customer meant to use LDAP but typed this wrong
hms-mirror.authentication.type=ldap
# But LDAP URL not configured

    Check that if ldap is set, the LDAP properties are also set: 

    hms-mirror.authentication.ldap.url=ldap://ldap.example.com:389
hms-mirror.authentication.ldap.base=dc=example,dc=com
hms-mirror.authentication.ldap.userDnPattern=uid={0},ou=users,dc=example,dc=com

    Working Directory Wrong¶

    # Customer changed this but directory doesn't exist or wrong permissions
hms-mirror.working-directory-base=/mnt/data/hms-mirror

    Verify the directory exists and is writable by the process user.

    Connection Configuration Issues¶

    Connection configs are stored in RocksDB and edited via the UI. Common mistakes:

    JDBC URI Format Errors¶

    Wrong Correct
    hive2://host:10000 jdbc:hive2://host:10000/default
    jdbc:hive2://host/ (no port) jdbc:hive2://host:10000/default
    Binary URI for HTTP-mode HS2 Add ;transportMode=http;httpPath=/cliservice

    Missing or Wrong Database in URI¶

    The database name after the last / matters: 

    jdbc:hive2://host:10000/default    ← connects to 'default' database
jdbc:hive2://host:10000/           ← may fail or use wrong database

    Wrong Platform Type¶

    Setting the wrong platform type (HDP vs CDP) can cause the engine to generate incompatible SQL. If migrations succeed but result in wrong behavior, check that the platform type matches the actual cluster.

    ConfigLite Issues¶

    ConfigLite controls migration behavior. These are edited via Configurations in the UI.

    ACID Tables Failing¶

    # Add this to ConfigLite to enable ACID table downgrade
acid:
  downgradeAcid: true

    Views Not Being Migrated¶

    Views are disabled by default in some configurations: 

    migrate:
  views: true    # Enable view migration

    Iceberg Conversion Not Working¶

    # Must be explicitly enabled
iceberg:
  enabled: true
  version: 2

    Warehouse Path Override Needed¶

    When source and target use different warehouse paths: 

    transfer:
  warehouse:
    managedDirectory: /warehouse/tablespace/managed/hive
    externalDirectory: /warehouse/tablespace/external/hive

    Dataset Configuration Issues¶

    Regex Pattern Not Matching Expected Tables¶

    HMS-Mirror uses Java regex syntax. Common mistakes:

    # Wrong: shell glob syntax
include:
  tables: "orders*"       # Matches nothing in Java regex (literal asterisk)

# Correct: Java regex syntax
include:
  tables: "orders.*"      # Matches orders, orders_2024, orders_details, etc.

    Database List Not Matching¶

    # This only migrates the 'sales' database
databases:
  - sales

# To migrate multiple:
databases:
  - sales
  - marketing
  - finance

# To migrate all databases (use with caution):
databases:
  - "*"

    YAML Parsing Errors¶

    Configuration exports/imports use YAML. If a YAML file is manually edited and fails to load:

    org.yaml.snakeyaml.parser.ParserException: mapping values are not allowed here

    Common YAML mistakes: - Tabs instead of spaces (YAML requires spaces) - Missing quotes around values with special characters (:, #, &, *) - Incorrect indentation level - Missing list dash (-) prefix

    Validate YAML before importing using an online YAML validator or: 

    python3 -c "import yaml; yaml.safe_load(open('config.yaml'))"
