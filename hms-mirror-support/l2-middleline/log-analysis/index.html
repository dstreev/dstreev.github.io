    Log Analysis¶

    The application log is the primary diagnostic tool at L2. This page teaches you to read and interpret it.

    Log Location and Rotation¶

    Item Value
    Primary log file ~/.hms-mirror/logs/hms-mirror.log
    Rotation trigger 100 MB file size
    Rolled files hms-mirror.YYYY-MM-DD.N.log.gz
    Retention 30 days of history
    Maximum total size 1 GB

    Log Format¶

    Each log line follows the pattern:

    TIMESTAMP  [THREAD]  LEVEL  LOGGER  - MESSAGE

    Example: 

    2026-02-26 10:15:33.421  [http-nio-8090-exec-3]  INFO  c.c.u.h.m.security.JwtAuthenticationFilter - Set authentication for user: alice with roles: [USER]

    Field Example Meaning
    Timestamp 2026-02-26 10:15:33.421 When the event occurred
    Thread http-nio-8090-exec-3 Web request thread 3
    Level INFO Log severity
    Logger c.c.u.h.m.security.JwtAuthenticationFilter Java class that logged this
    Message Set authentication for user: alice What happened

    Log Levels and What They Mean¶

    Level When to care
    ERROR Something failed — always investigate
    WARN Something unexpected happened but continued — may need attention
    INFO Normal operation messages — confirm startup, connection, job progress
    DEBUG Detailed trace (only if debug logging was enabled)

    For normal troubleshooting, focus on ERROR and WARN lines.

    Startup Sequence — What to Look For¶

    A healthy startup log looks like:

    INFO  Starting Mirror ...
INFO  Starting Spring Boot application with profile: default
INFO  RocksDB opened successfully at: /home/user/.hms-mirror/data
INFO  Keystore loaded from: /home/user/.hms-mirror/security/hms-mirror.p12
INFO  JWT configuration initialized
INFO  Security configuration loaded
INFO  Started Mirror in 8.432 seconds (JVM running for 9.1)
INFO  HMS-Mirror Server started on port: 8090

    Startup Failure Patterns¶

    # Find startup errors quickly
grep -E "ERROR|WARN|Exception|Failed|Could not" ~/.hms-mirror/logs/hms-mirror.log | head -20
    Log Pattern Root Cause
    RocksDB: Could not acquire lock Another instance is running, or stale lock file
    RocksDB: Corruption detected Database integrity failure — escalate L3
    Keystore not found at path Security keystore file missing
    Failed to load keystore Wrong keystore password
    Port 8090 was already in use Port conflict
    UnsupportedClassVersionError Wrong Java version
    Address already in use Port conflict
    OutOfMemoryError Insufficient heap at startup

    Authentication Log Patterns¶

    Successful Login¶

    INFO  c.c.u.h.m.security.AuthServiceImpl - User alice logged in successfully
INFO  c.c.u.h.m.security.jwt.JwtAuthenticationFilter - Set authentication for user: alice with roles: [USER, ADMIN]

    Failed Login¶

    WARN  c.c.u.h.m.security.AuthServiceImpl - Authentication failed for user: alice — invalid credentials
WARN  c.c.u.h.m.security.AuthServiceImpl - Failed login attempt 3/5 for user: alice

    Account Locked¶

    WARN  c.c.u.h.m.security.AuthServiceImpl - Account locked for user: alice after 5 failed attempts

    Rate Limit Hit¶

    WARN  c.c.u.h.m.security.LoginRateLimitFilter - Rate limit exceeded for IP: 10.0.0.1

    Job / Migration Log Patterns¶

    Job Started¶

    INFO  c.c.u.h.m.service.JobService - Starting job: <job-id> strategy: SQL
INFO  c.c.u.h.m.datastrategy.SqlDataStrategy - Processing database: sales_db (12 tables)

    Table Success¶

    INFO  c.c.u.h.m.datastrategy - Table orders: SUCCESS

    Table Failure¶

    ERROR c.c.u.h.m.datastrategy - Table orders: FAILED
ERROR c.c.u.h.m.datastrategy - Cause: org.apache.hive.jdbc.HiveDriver: Access denied

    Connection Error During Job¶

    ERROR c.c.u.h.m.service - Cannot connect to source: jdbc:hive2://host:10000
ERROR c.c.u.h.m.service - java.sql.SQLException: Unable to connect

    Connection Log Patterns¶

    Successful Connection Test¶

    INFO  c.c.u.h.m.service.ConnectionService - Connection test successful: MySourceCluster
INFO  c.c.u.h.m.service.ConnectionService - Hive version: 3.1.3000.7.1

    Connection Failure¶

    ERROR c.c.u.h.m.service.ConnectionService - Connection test failed: MySourceCluster
ERROR c.c.u.h.m.service.ConnectionService - java.sql.SQLException: Could not establish connection
ERROR c.c.u.h.m.service.ConnectionService - Caused by: java.net.ConnectException: Connection refused

    Enabling Debug Logging¶

    For deeper investigation, enable debug-level logging. Add to web.properties:

    logging.level.com.cloudera.utils.hms=DEBUG

    Or pass at startup: 

    java -jar hmsm-server-*.jar \
  --logging.level.com.cloudera.utils.hms=DEBUG \
  --logging.level.org.apache.hive=WARN

    Log Volume

    DEBUG logging generates very high log volume. Only enable temporarily and disable after collecting the needed information. The 1GB log cap will fill quickly.

    Useful Log Analysis Commands¶

    # Last 100 lines
tail -100 ~/.hms-mirror/logs/hms-mirror.log

# All errors and warnings
grep -E "(ERROR|WARN)" ~/.hms-mirror/logs/hms-mirror.log

# All errors from a specific job (replace with actual job ID)
grep "<job-id>" ~/.hms-mirror/logs/hms-mirror.log

# Connection failures only
grep -i "connection.*fail\|fail.*connection\|ConnectException" ~/.hms-mirror/logs/hms-mirror.log

# Authentication events
grep "c.c.u.h.m.security" ~/.hms-mirror/logs/hms-mirror.log

# Startup sequence only
grep "Starting\|Started\|RocksDB\|Keystore\|JWT\|port" ~/.hms-mirror/logs/hms-mirror.log | head -30

# Stack trace (find "Caused by" lines)
grep -A 5 "Exception\|Caused by" ~/.hms-mirror/logs/hms-mirror.log | head -50
