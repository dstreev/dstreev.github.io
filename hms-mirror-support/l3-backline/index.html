    L3 Backline Support¶

    Audience: Senior engineers with deep Hadoop ecosystem knowledge, JVM internals familiarity, and direct access to customer environments when needed.

    Prerequisites: Complete familiarity with L1 and L2 documentation. L3 work assumes L2 has already diagnosed the problem category and ruled out configuration errors.

    Your job: Solve problems that require deep technical knowledge: database-level operations, security infrastructure, JVM tuning, incident response, and engineering escalation packaging.

    What L3 Handles¶

    • RocksDB diagnostics, recovery, and backup/restore
    • TLS/SSL certificate chain issues and replacement
    • Kerberos deep debugging (KDC, keytab, realm configuration)
    • JVM performance profiling and heap analysis
    • Security incident response (account compromise, audit)
    • Encryption keystore recovery procedures
    • API-level debugging with authentication tokens
    • Engineering escalation package preparation
    • Production data recovery coordination

    What L3 Does NOT Handle¶

    • Direct database manipulation without explicit procedure
    • Code changes (escalate to Engineering)
    • Customer cluster administration (guide customer, don't do it for them)

    L3 Diagnostic Environment¶

    Before starting any L3 work, establish a safe working context:

    # Know exactly where the working directory is
echo "Working dir: ${HMS_MIRROR_WORKING_DIR:-$HOME/.hms-mirror}"

# Check disk space before any operations
df -h ~/.hms-mirror/

# Get current server status
curl -s http://localhost:8090/hms-mirror/api/v1/system/health | python3 -m json.tool

# Backup the data directory before any changes
tar -czf hms-mirror-data-backup-$(date +%Y%m%d-%H%M).tar.gz ~/.hms-mirror/

    Always Backup First

    Before any RocksDB repair, keystore operation, or configuration change at L3, create a backup. Many L3 operations are irreversible.
