    TLS/SSL Deep Dive¶

    This page covers HTTPS configuration, certificate issues, and trust chain debugging for HMS-Mirror.

    TLS Architecture¶

    HMS-Mirror has two distinct TLS contexts:

    Context Purpose
    Server TLS HTTPS for the HMS-Mirror web interface (inbound)
    Client TLS Trust store for connecting to Hive/HDFS clusters with TLS (outbound)

    Both can have problems independently.

    Server TLS (HTTPS for the UI)¶

    Configuration¶

    TLS is configured in ~/.hms-mirror/settings/tls.properties:

    server.ssl.enabled=true
server.ssl.key-store=~/.hms-mirror/certificates/hms-mirror.p12
server.ssl.key-store-password=changeit
server.ssl.key-store-type=PKCS12
server.ssl.key-alias=hms-mirror

    And optionally in web.properties: 

    server.port=8443

    Verify TLS Configuration¶

    # Test HTTPS connectivity
curl -vk https://localhost:8443/hms-mirror/api/v1/system/health

# View the certificate being served
openssl s_client -connect localhost:8443 -servername localhost < /dev/null 2>&1 | \
  openssl x509 -noout -text -in /dev/stdin

# Check certificate expiry
openssl s_client -connect localhost:8443 < /dev/null 2>&1 | \
  openssl x509 -noout -dates

    Common TLS Startup Errors¶

    Error Cause Fix
    java.io.FileNotFoundException: keystore Keystore file path wrong Fix path in tls.properties
    UnrecoverableKeyException Wrong keystore password Fix password in tls.properties
    java.security.KeyStoreException: PKCS12 Keystore type mismatch Ensure type=PKCS12
    javax.net.ssl.SSLHandshakeException Client doesn't trust the cert Customer browser needs to trust the CA

    Creating a Self-Signed Certificate (Development)¶

    # Generate a self-signed certificate (1 year validity)
keytool -genkeypair \
  -alias hms-mirror \
  -keyalg RSA \
  -keysize 2048 \
  -storetype PKCS12 \
  -keystore ~/.hms-mirror/certificates/hms-mirror.p12 \
  -validity 365 \
  -storepass changeit \
  -dname "CN=hms-mirror.example.com, OU=Support, O=Cloudera, L=Santa Clara, ST=CA, C=US"

    Client TLS (Connecting to Secured Hive/Knox)¶

    When HMS-Mirror connects to Hive clusters with TLS enabled, it needs to trust the cluster's certificate.

    Adding a Trusted Certificate¶

    Option 1: Add to the JVM trust store

    # Find Java installation
JAVA_HOME=$(dirname $(dirname $(readlink -f $(which java))))

# Import the cluster CA certificate
keytool -importcert \
  -alias source-cluster-ca \
  -file /path/to/cluster-ca.crt \
  -keystore $JAVA_HOME/lib/security/cacerts \
  -storepass changeit \
  -noprompt

    Option 2: Create a custom trust store for HMS-Mirror

    # Create trust store with cluster CA
keytool -importcert \
  -alias source-cluster-ca \
  -file /path/to/cluster-ca.crt \
  -keystore ~/.hms-mirror/certificates/truststore.jks \
  -storepass changeit \
  -noprompt

# Configure HMS-Mirror to use it (in web.properties or startup args)
-Djavax.net.ssl.trustStore=~/.hms-mirror/certificates/truststore.jks
-Djavax.net.ssl.trustStorePassword=changeit

    JDBC URI with SSL¶

    For HiveServer2 with TLS: 

    jdbc:hive2://host:10000/default;ssl=true;sslTrustStore=/path/to/truststore.jks;trustStorePassword=changeit

    Certificate Expiry¶

    Certificates have expiry dates. When they expire, connections suddenly start failing with SSL errors.

    Check all certificates HMS-Mirror depends on:

    # Check the server certificate
openssl s_client -connect localhost:8443 < /dev/null 2>&1 | \
  openssl x509 -noout -dates

# Check a remote cluster certificate
openssl s_client -connect hiveserver2.example.com:10000 < /dev/null 2>&1 | \
  openssl x509 -noout -dates

# Find certificates in the working directory
find ~/.hms-mirror/certificates/ -name "*.crt" -o -name "*.pem" | \
  xargs -I{} openssl x509 -noout -dates -in {}

    Knox SSL Troubleshooting¶

    Knox uses mutual TLS in some configurations. Common issues:

    Error: PKIX path building failed: unable to find valid certification path to requested target

    This means HMS-Mirror doesn't trust the Knox certificate. Solutions: 1. Import the Knox CA certificate into HMS-Mirror's trust store (see above) 2. For development: use -k in curl tests, but do NOT use ssl=false in production JDBC URIs

    Collecting TLS Diagnostic Info for Escalation¶

    # Certificate chain from the server
openssl s_client -connect localhost:8443 -showcerts < /dev/null 2>&1

# SSL/TLS handshake debug (verbose, generates a lot of output)
java -Djavax.net.debug=ssl:handshake \
  -jar hmsm-server-*.jar 2>&1 | \
  grep -E "ServerHello|Certificate|Finished|Exception" | head -50
