    HMS-Mirror Configuration Validation Rules¶

    This document describes the validation rules that HMS-Mirror enforces before executing a migration job. These rules ensure that the combination of data strategy, configuration options, connection settings, and platform types form a valid and safe migration plan.

    Concepts¶

    • LEFT: The source cluster/environment
    • RIGHT: The target cluster/environment
    • Data Strategy: The migration approach (SCHEMA_ONLY, SQL, EXPORT_IMPORT, HYBRID, LINKED, STORAGE_MIGRATION, DUMP, COMMON, CONVERT_LINKED, ICEBERG_CONVERSION)
    • ACID tables: Hive transactional tables (managed tables in Hive 3+)
    • Legacy Hive: Hive 1.x/2.x (pre-transactional)
    • distcp: Hadoop Distributed Copy, used for large-scale data movement
    • Warehouse Plan: Defines external and managed warehouse directory mappings for a database
    • HCFS Profile: Hadoop-Compatible File System profile defining namespace and storage configuration
    • Target Namespace: The root filesystem namespace (e.g., hdfs://namenode, s3a://bucket, ofs://vol) for the migration target

    1. Required Components¶

    1.1 LEFT Connection Required¶

    • Rule: A LEFT (source) connection must always be defined.
    • Error: MISSING_CONFIGURATION: LEFT(Source) connection

    1.2 RIGHT Connection Conditionally Required¶

    • Rule: A RIGHT (target) connection is required UNLESS:
    • The strategy is STORAGE_MIGRATION or DUMP
    • OR ACID in-place downgrade is enabled (migrateACID.inplace=true AND migrateACID.downgrade=true) with strategy SQL, EXPORT_IMPORT, or HYBRID
    • Error: MISSING_CONFIGURATION: RIGHT(Target) connection

    1.3 Dataset Required¶

    • Rule: A dataset must be defined with at least one database specified.
    • Error: MISSING_CONFIGURATION: Dataset or MISSING_CONFIGURATION: Databases

    2. JDBC Driver Validation¶

    2.1 HiveServer2 Driver¶

    • Rule: When a connection has an HS2 URI configured, the corresponding Hive JDBC driver must be available on the classpath.
    • Applies to: Both LEFT and RIGHT connections.
    • Error: DRIVER_NOT_FOUND: <environment>: HiveServer2 driver not found: <driver>. Please download via Utilities -> Drivers -> Hive Drivers

    2.2 Metastore Direct Driver¶

    • Rule: When a connection has Metastore Direct enabled with a URI configured, the corresponding RDBMS JDBC driver (MySQL, MariaDB, Postgres, or Oracle) must be installed.
    • Applies to: Both LEFT and RIGHT connections.
    • Error: DRIVER_NOT_FOUND: <environment>: Metastore JDBC driver not installed: <type>. Please install via Utilities -> Drivers -> Metastore RDBMS Drivers

    3. HCFS Profile Validation¶

    3.1 Target Namespace Profile¶

    • Rule: A target namespace must be resolvable from an HCFS profile. Resolution order:
    • Explicit targetNamespaceProfileKey on the job
    • RIGHT connection's hcfsConfigProfileKey (for most strategies)
    • Exception: LINKED strategy uses LEFT connection's namespace; no target namespace needed.
    • Error: TARGET_NAMESPACE_PROFILE_REQUIRED

    3.2 HCFS Profile Must Exist with Namespace¶

    • Rule: When an HCFS profile key is referenced, the profile must exist and must have a defaultNamespace configured.
    • Errors: HCFS_PROFILE_NOT_FOUND: <profileKey> or HCFS_PROFILE_MISSING_NAMESPACE: <profileKey>

    3.3 Intermediate Storage Profile¶

    • Rule: When an intermediate storage profile key is specified on the job, the referenced profile must exist.
    • Error: HCFS_PROFILE_NOT_FOUND: <profileKey>

    4. Platform Compatibility Rules¶

    4.1 No Hive Downgrades¶

    • Rule: The RIGHT cluster's Hive version must be greater than or equal to the LEFT cluster's Hive version. Migrating from Hive 3 to Hive 1/2 is not supported.
    • Error: HIVE_DOWNGRADE_REQUESTED

    4.2 Non-Legacy to Legacy Prohibited¶

    • Rule: Migrating from a non-legacy Hive (Hive 3+) source to a legacy Hive (Hive 1/2) target is not supported.
    • Exception: Does not apply when ACID in-place downgrade is enabled (single-cluster operation).
    • Error: NON_LEGACY_TO_LEGACY

    4.3 Legacy Hive Cannot Be CONVERT_LINKED Target¶

    • Rule: The CONVERT_LINKED strategy does not support legacy Hive as the RIGHT (target) cluster.
    • Error: LEGACY_HIVE_RIGHT_CLUSTER

    5. Strategy-Specific Rules¶

    5.1 ACID is Not a Top-Level Strategy¶

    • Rule: The ACID strategy cannot be selected directly. Use HYBRID or SQL with the migrateACID option enabled.
    • Error: ACID_NOT_TOP_LEVEL_STRATEGY

    5.2 DUMP Strategy¶

    • Rule: Translation type is forced to RELATIVE. Data movement strategy is forced to NA. Execution is forced to dry-run mode.

    5.3 STORAGE_MIGRATION Strategy¶

    • Rule: Not available for Legacy Hive sources.

    • Error: STORAGE_MIGRATION_NOT_AVAILABLE_FOR_LEGACY

    • Rule: The LEFT connection must have Metastore Direct configured (required for pulling table/partition location details).

    • Error: METASTORE_DIRECT_NOT_DEFINED_OR_CONFIGURED: LEFT

    • Rule: Translation type is forced to ALIGNED.

    • Rule: When source platform is HDP3, forceExternalLocation is automatically set to true, and ACID transfers without distcp are prohibited.

    • Error: HIVE3_ON_HDP_ACID_TRANSFERS

    • Rule: Cannot use distcp with ACID tables when createArchive is also enabled.

    • Error: STORAGE_MIGRATION_ACID_ARCHIVE_DISTCP

    5.4 SCHEMA_ONLY Strategy¶

    • Rule: ACID downgrade is not supported with SCHEMA_ONLY (requires SQL transformations).

    • Error: ACID_DOWNGRADE_SCHEMA_ONLY

    • Rule: With RELATIVE translation type, data movement strategy is forced to MANUAL (not SQL).

    • Rule: With ALIGNED translation type, data movement strategy is forced to DISTCP.

    • Warning (DISTCP_W_RELATIVE): Using distcp with RELATIVE translations only inspects database-level locations, not individual table/partition locations.

    • Warning (RELATIVE_MANUAL): RELATIVE with MANUAL may not correctly migrate non-standard table/partition locations.

    5.5 SQL and HYBRID Strategies¶

    • Rule: Data movement strategy is forced to SQL.
    • Rule: distcp with ACID tables is only valid when downgrading (migrateACID.downgrade=true).
    • Error: SQL_DISTCP_ONLY_W_DA_ACID

    5.6 EXPORT_IMPORT Strategy¶

    • Rule: Data movement strategy is forced to NA.

    5.7 LINKED Strategy¶

    • Rule: ACID table migration is not supported with LINKED (cannot link transactional tables).

    • Error: LINKED_NO_ACID_SUPPORT

    • Rule: Intermediate storage is not valid with LINKED.

    • Error: INTERMEDIATE_STORAGE_WITH_LINKED

    • Rule: Target namespace/HCFS profile is removed (LINKED uses LEFT namespace).

    5.8 HDP3 Target Warning¶

    • Rule: When the RIGHT cluster is HDP with Hive 3, forceExternalLocation is automatically set to true.
    • Warning: HDP3_HIVE

    6. ACID Table Migration Rules¶

    6.1 Valid ACID Strategies¶

    • Rule: ACID migration (migrateACID.on=true) is only valid with: SCHEMA_ONLY, DUMP, SQL, EXPORT_IMPORT, HYBRID, STORAGE_MIGRATION.
    • Error: VALID_ACID_STRATEGIES

    6.2 ACID In-Place Downgrade¶

    • Rule: In-place ACID downgrade (migrateACID.inplace=true) is only valid with the SQL strategy.

    • Error: VALID_ACID_DA_IP_STRATEGIES

    • Rule: In-place downgrade requires a target namespace to be defined.

    • Error: TARGET_NAMESPACE_NOT_DEFINED

    • Rule: In-place downgrade cannot use intermediate storage.

    • Error: INTERMEDIATE_STORAGE_WITH_DA_IP

    • Rule: In-place downgrade cannot use distcp.

    • Error: DISTCP_W_DA_IP_ACID

    • Rule: In-place downgrade only works on non-legacy Hive (Hive 3+).

    • Error: DA_IP_NON_LEGACY

    7. Iceberg Conversion Rules¶

    7.1 Strategy Restriction¶

    • Rule: Iceberg conversion is only available with STORAGE_MIGRATION or SQL strategies. If enabled with other strategies, it is automatically disabled.

    7.2 Data Movement Restriction¶

    • Rule: Iceberg conversion cannot use distcp. Only SQL data movement is supported.

    • Error: ICEBERG_CONVERSION_DISTCP

    • Rule: When Iceberg conversion is enabled with STORAGE_MIGRATION, data movement strategy is forced to SQL.

    8. Sync Mode Rules¶

    8.1 Valid Sync Strategies¶

    • Rule: The sync option is only valid with: SCHEMA_ONLY, LINKED, SQL, EXPORT_IMPORT, HYBRID, COMMON.
    • Error: VALID_SYNC_STRATEGIES

    8.2 Sync Behavior¶

    • Rule: When sync is enabled with SCHEMA_ONLY or LINKED, disaster recovery mode is automatically enabled.

    8.3 Sync with Table Filters¶

    • Warning (SYNC_TBL_FILTER): When sync is combined with table filters, synchronization is bi-directional only for tables matching the filter on both sides.

    9. Read-Only Mode Rules¶

    9.1 Valid Read-Only Strategies¶

    • Rule: Read-only mode is only valid with: SCHEMA_ONLY, LINKED, COMMON, SQL.
    • Error: RO_VALID_STRATEGIES

    10. Database Configuration Rules¶

    10.1 dbRename and dbPrefix Are Mutually Exclusive¶

    • Rule: You cannot set both dbRename and dbPrefix on the same database specification.
    • Error: CONFLICTING_PROPERTIES: dbRename, dbPrefix

    11. Distcp and Warehouse Plan Rules¶

    11.1 Aligned Distcp Requires Warehouse Plans¶

    • Rule: When using ALIGNED translation type with distcp data movement, warehouse plans must be defined for the databases.
    • Error: ALIGN_LOCATIONS_WITHOUT_WAREHOUSE_PLANS

    11.2 Distcp with Table Filters¶

    • Warning (DISTCP_W_TABLE_FILTERS): When using distcp with table filters, separate distcp plans are built per table. This may affect completeness of data transfer.

    11.3 SQL ACID Downgrade with Distcp¶

    • Rule: Using SQL strategy with ACID downgrade and distcp requires external warehouse directory (-ewd) to be specified.
    • Error: SQL_ACID_DA_DISTCP_WO_EXT_WAREHOUSE

    12. Ozone-Specific Rules¶

    12.1 Volume Name Length¶

    • Rule: When the target namespace uses ofs:// (Ozone), volume names derived from warehouse directory paths must be at least 3 characters long.
    • Applies to: Both external and managed warehouse directories in each database specification.
    • Error: OZONE_VOLUME_NAME_TOO_SHORT

    13. Target Namespace Rules¶

    13.1 Namespace Required for Most Strategies¶

    • Rule: A resolvable target namespace is required for all strategies except DUMP and LINKED.
    • Error: TARGET_NAMESPACE_NOT_DEFINED

    14. Configuration Auto-Adjustments¶

    The validation process also makes automatic adjustments to invalid-but-recoverable configuration combinations. These are logged as configuration adjustment messages but do not fail the validation.

    Scenario Auto-Adjustment
    Iceberg conversion enabled with unsupported strategy Iceberg conversion disabled
    DUMP with non-RELATIVE translation type Translation type set to RELATIVE
    DUMP with execute enabled Forced to dry-run
    SCHEMA_ONLY + RELATIVE + SQL data movement Data movement set to MANUAL
    SCHEMA_ONLY + ALIGNED + non-DISTCP data movement Data movement set to DISTCP
    SQL/HYBRID with non-SQL data movement Data movement set to SQL
    EXPORT_IMPORT with non-NA data movement Data movement set to NA
    STORAGE_MIGRATION with non-ALIGNED translation Translation set to ALIGNED
    STORAGE_MIGRATION + Iceberg + non-SQL data movement Data movement set to SQL
    STORAGE_MIGRATION with NA data movement Data movement set to DISTCP
    LINKED with target namespace defined Target namespace removed
    Sync with SCHEMA_ONLY or LINKED Disaster recovery mode enabled
    HDP3 target cluster forceExternalLocation set to true

    Rule Evaluation Order¶

    1. Required components check — LEFT connection, dataset, databases (fail-fast if missing)
    2. Driver validation — Verify JDBC drivers for both HS2 and Metastore Direct
    3. HCFS profile validation — Target namespace and intermediate storage profiles
    4. Configuration alignment — Auto-adjust recoverable misconfigurations
    5. Strategy-specific validation — Rules specific to the selected data strategy
    6. Cross-cutting validation — Ozone volumes, ACID rules, sync rules, read-only rules
    7. Final error check — Any accumulated errors from sub-validators fail the validation
