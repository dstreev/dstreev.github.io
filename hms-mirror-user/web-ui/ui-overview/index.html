    Web UI Overview¶

    This guide introduces the HMS-Mirror web interface, its navigation structure, and common UI patterns.

    Accessing the Web UI¶

    After starting HMS-Mirror, open your browser to:

    http://localhost:8090/hms-mirror

    Replace localhost and 8090 with your server's hostname and configured port.

    Dashboard¶

    The dashboard is your landing page, providing an overview of your HMS-Mirror setup.

    Navigation¶

    Main Menu¶

    The navigation menu provides access to all major sections.

    Menu Item Description Path
    Dashboard Overview and status /hms-mirror
    Connections Cluster connections /hms-mirror/connections
    Configurations Migration settings /hms-mirror/config
    Datasets Database/table selections /hms-mirror/datasets
    Jobs Build and manage jobs /hms-mirror/jobs
    Reports View migration reports /hms-mirror/reports
    Settings Application settings /hms-mirror/settings

    Sub-Navigation¶

    Some sections have sub-pages:

    Jobs - Build - Create and manage jobs - Runtime - Running job status

    Settings - Preferences - User preferences - LLM - AI integration settings - RocksDB - Storage management

    Common UI Patterns¶

    Cards¶

    Items are displayed as cards with consistent layout.

    ┌─────────────────────────────────────────────────────────┐
│ ┌─────┐                                                 │
│ │Icon │  Item Name                              [Edit]  │
│ └─────┘  Description or subtitle                [Delete]│
│                                                         │
│  Key Information                                        │
│  • Property 1: Value                                    │
│  • Property 2: Value                                    │
│                                                         │
│  [Action Button 1]  [Action Button 2]  [···]           │
└─────────────────────────────────────────────────────────┘

    Card Actions¶

    Common actions on cards:

    Icon Action Description
    ✏️ Edit Modify the item
    📋 Copy Duplicate the item
    🗑️ Delete Remove the item
    ▶️ Run Execute (jobs)
    ✓ Validate Check for errors
    ✨ AI Analysis Get AI recommendations
    📊 Summary View details
    📥 Export Download as file

    Lists and Grids¶

    Icon Meaning

    Interactive Features¶

    Drag and Drop¶

    Some interfaces support drag and drop: - Reordering items - Uploading files - Moving between lists

    Context Menus¶

    Right-click for additional options (where supported).

    Keyboard Shortcuts¶

    Shortcut Action
    Esc Close dialog/cancel
    Enter Submit form
    Tab Navigate fields
    Ctrl+S Save (in forms)

    Responsive Design¶

    The UI adapts to different screen sizes:

    Desktop (>1024px) - Full sidebar navigation - Multi-column card layouts - Side-by-side form columns

    Tablet (768px-1024px) - Collapsible navigation - Two-column layouts - Stacked form sections

    Mobile (<768px) - Hamburger menu - Single-column layouts - Full-width forms

    Accessibility¶

    HMS-Mirror follows accessibility best practices:

    • Keyboard navigation - All features accessible via keyboard
    • Screen reader support - Proper ARIA labels
    • Color contrast - WCAG compliant colors
    • Focus indicators - Visible focus states

    Customization¶

    Theme¶

    The UI uses a consistent color theme: - Primary: Blue (#3B82F6) - Success: Green (#10B981) - Warning: Yellow (#F59E0B) - Error: Red (#EF4444) - Neutral: Gray scale

    Preferences¶

    User preferences (Settings → Preferences): - Items per page - Default view (grid/list) - Notification settings
