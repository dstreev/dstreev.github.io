    Troubleshooting Guide¶

    This guide helps you diagnose and resolve common issues with HMS-Mirror.

    Connection Issues¶

    Cannot Connect to HiveServer2¶

    Symptoms 

    Error: Could not establish connection to jdbc:hive2://...
java.sql.SQLException: Unable to connect

    Possible Causes and Solutions

    Cause Solution
    Service not running Verify HiveServer2 is running on the cluster
    Wrong host/port Check hostname and port (default: 10000 binary, 10001 HTTP)
    Firewall blocking Check firewall rules, security groups
    Wrong transport mode Match transportMode (binary vs http) to server config
    SSL mismatch Enable/disable SSL to match server

    Diagnostic Steps

    1. Test basic connectivity: 

      telnet your-host.example.com 10000

    2. Test with beeline: 

      beeline -u "jdbc:hive2://your-host:10000/default"

    3. Check server logs on the cluster

    Kerberos Authentication Failures¶

    Symptoms 

    Error: GSS initiate failed
javax.security.auth.login.LoginException: Unable to obtain password

    Possible Causes and Solutions

    Cause Solution
    No valid ticket Run kinit to obtain a ticket
    Expired ticket Run kinit to renew
    Wrong principal Verify the service principal
    Clock skew Sync time with KDC
    krb5.conf issues Verify /etc/krb5.conf configuration

    Diagnostic Steps

    1. Check for valid ticket: 

      klist

    2. Obtain new ticket: 

      kinit -kt /path/to/keytab principal@REALM
# or interactively
kinit principal@REALM

    3. Test principals: 

      kvno hive/hostname@REALM

    Knox Gateway Issues¶

    Symptoms 

    Error: HTTP 401 Unauthorized
Error: HTTP 403 Forbidden
Error: SSL handshake failed

    Possible Causes and Solutions

    Cause Solution
    Wrong credentials Verify Knox username/password
    Wrong httpPath Check Knox topology path
    Certificate issues Configure trust store correctly
    Knox service down Verify Knox is running

    Example Knox URI 

    jdbc:hive2://knox.example.com:8443/;ssl=true;transportMode=http;httpPath=gateway/cdp-proxy-api/hive

    Metastore Direct Connection Failures¶

    Symptoms 

    Error: Cannot connect to metastore database
Error: Access denied for user 'hive'@'host'

    Possible Causes and Solutions

    Cause Solution
    Wrong JDBC URL Verify database host, port, name
    Wrong credentials Check username and password
    Missing driver Ensure JDBC driver is in aux_libs
    Network blocked Check database firewall rules
    Permissions Grant SELECT on metastore tables

    Required Drivers

    Place in ~/.hms-mirror/aux_libs/: - MySQL: mysql-connector-java-8.0.x.jar - PostgreSQL: postgresql-42.x.x.jar - Oracle: ojdbc8.jar

    Migration Failures¶

    Table Creation Failures¶

    Symptoms 

    Error: Error creating table
Error: Table already exists
Error: Permission denied

    Solutions by Error

    Table Already Exists - Enable table overwrite in configuration - Drop existing table manually - Use a different target database name

    Permission Denied - Verify user has CREATE TABLE permission - Check database permissions - Verify warehouse directory permissions

    Invalid Location - Ensure storage path exists - Check path permissions - Verify namespace is accessible

    Data Movement Failures¶

    Symptoms 

    Error: Insert failed
Error: EXPORT failed
Error: File not found

    SQL Strategy Failures

    Error Cause Solution
    Out of memory Table too large Switch to EXPORT_IMPORT
    Timeout Query running too long Increase timeout, use EXPORT
    Type conversion Incompatible types Check column types

    EXPORT_IMPORT Strategy Failures

    Error Cause Solution
    No space Insufficient storage Clear space, use different path
    Permission denied Can't write to export path Check directory permissions
    Import failed Incompatible Hive versions Check version compatibility

    ACID Table Issues¶

    Symptoms 

    Error: Cannot export transactional table
Error: ACID operations not supported
Error: Compaction required

    Solutions

    Issue Solution
    Export fails Use EXPORT_IMPORT strategy
    Wrong Hive version Ensure compatible Hive 3.x versions
    Compaction needed Run compaction before migration
    Bucket mismatch Adjust bucket threshold settings

    ACID Configuration

    If ACID tables are problematic: 

    acid:
  downgradeAcid: true  # Convert to external

    Partition Issues¶

    Symptoms 

    Error: Too many partitions
Error: Partition not found
Error: Dynamic partition strict mode

    Solutions

    Issue Solution
    Partition limit exceeded Increase limit or use EXPORT_IMPORT
    Missing partitions Verify source partitions exist
    Strict mode Set hive.exec.dynamic.partition.mode=nonstrict

    Configuration for Large Partitioned Tables

    hybrid:
  sqlPartitionLimit: 500
  exportImportPartitionLimit: 1000

optimizations:
  overrides:
    - key: hive.exec.dynamic.partition.mode
      rightValue: nonstrict
    - key: hive.exec.max.dynamic.partitions
      rightValue: "10000"

    Performance Issues¶

    Slow Migrations¶

    Symptoms - Jobs taking much longer than expected - Timeouts during migration - Resource exhaustion

    Diagnostic Questions

    1. How large are the tables?
    2. How many partitions?
    3. What strategy is being used?
    4. What are cluster resources like?

    Solutions by Cause

    Large Tables with SQL Strategy - Switch to EXPORT_IMPORT - Use HYBRID with appropriate thresholds - Increase parallelism

    Many Small Tables - Increase parallelism - Ensure metadata operations are fast - Check Metastore performance

    Network Bottleneck - Check network bandwidth - Run HMS-Mirror closer to clusters - Use DISTCP for large data movement

    Memory Issues¶

    Symptoms 

    Error: Java heap space
Error: OutOfMemoryError
Error: GC overhead limit exceeded

    Solutions

    Increase HMS-Mirror memory: 

    java -Xmx8g -Xms4g -jar hms-mirror-server.jar

    Reduce job scope: - Process fewer databases per job - Split large datasets - Lower parallelism settings

    Query Timeouts¶

    Symptoms 

    Error: Query timed out
Error: SocketTimeoutException

    Solutions

    Increase timeouts: 

    hiveServer2:
  connectionTimeout: 300000  # 5 minutes
  queryTimeout: 3600000      # 1 hour

    Or switch strategy: - Use EXPORT_IMPORT instead of SQL - Break large tables into smaller jobs

    RocksDB Issues¶

    Database Corruption¶

    Symptoms 

    Error: RocksDB corruption detected
Error: Cannot open database

    Solution

    1. Stop HMS-Mirror
    2. Backup the RocksDB directory: 
      mv ~/.hms-mirror/rocksdb ~/.hms-mirror/rocksdb.backup
    3. Restart HMS-Mirror (creates fresh database)
    4. Re-import configurations from backups

    Disk Space¶

    Symptoms 

    Error: No space left on device
Error: Cannot write to RocksDB

    Solutions

    1. Clear old data: 

      # Check disk usage
du -sh ~/.hms-mirror/rocksdb

    2. Move to larger disk: 

      mv ~/.hms-mirror/rocksdb /larger/disk/rocksdb
# Update application.properties
hms-mirror.rocksdb.path=/larger/disk/rocksdb

    Permission Issues¶

    Symptoms 

    Error: Permission denied: rocksdb
Error: Cannot create lock file

    Solution 

    chmod -R 755 ~/.hms-mirror/rocksdb
chown -R $USER ~/.hms-mirror/rocksdb

    Web UI Issues¶

    UI Not Loading¶

    Symptoms - Blank page - Connection refused - 404 errors

    Solutions

    Issue Solution
    Server not running Start HMS-Mirror server
    Wrong port Check configured port (default: 8090)
    Context path wrong Use /hms-mirror path
    Browser cache Clear browser cache, hard refresh

    Verify Server Running 

    curl http://localhost:8090/hms-mirror/api/v1/health

    API Errors¶

    Symptoms 

    Error 500: Internal Server Error
Error 400: Bad Request
Error 404: Not Found

    Diagnostic Steps

    1. Check server logs: 

      tail -f ~/.hms-mirror/logs/hms-mirror.log

    2. Enable debug logging: 

      logging.level.com.cloudera.utils.hms=DEBUG

    3. Test API directly: 

      curl -X GET http://localhost:8090/hms-mirror/api/v1/connections

    Log Analysis¶

    Log Locations¶

    Log Location
    Application ~/.hms-mirror/logs/hms-mirror.log
    Reports ~/.hms-mirror/reports/<job-id>/
    Console Standard output/error

    Enabling Debug Logging¶

    Edit application.properties: 

    logging.level.com.cloudera.utils.hms=DEBUG
logging.level.org.apache.hive=DEBUG

    Or at startup: 

    java -jar hms-mirror-server.jar --logging.level.com.cloudera.utils.hms=DEBUG

    Common Log Patterns¶

    Connection Success 

    INFO - Connected to HiveServer2: jdbc:hive2://...
INFO - Metastore connection established

    Connection Failure 

    ERROR - Failed to connect: java.sql.SQLException
WARN - Retrying connection...

    Migration Progress 

    INFO - Processing database: sales_db
INFO - Migrating table: orders (1/45)
INFO - Table orders: SUCCESS

    Migration Error 

    ERROR - Failed to migrate table: orders
ERROR - Cause: Partition limit exceeded

    Getting Help¶

    Information to Collect¶

    When reporting issues, gather:

    1. HMS-Mirror version 

      java -jar hms-mirror-server.jar --version

    2. Java version 

      java -version

    3. Configuration (sanitized - remove passwords)

    4. Log files with debug enabled

    5. Error messages (complete stack trace)

    6. Environment details

    7. Source cluster type and version
    8. Target cluster type and version
    9. Network topology

    Resources¶

    • Documentation: This guide and related docs
    • GitHub Issues: Report bugs and request features
    • Logs: Check application logs for detailed errors

    Self-Help Checklist¶

    Before seeking help:

    • Checked logs for detailed error messages
    • Verified network connectivity to clusters
    • Tested connections individually
    • Ran validation on job
    • Tried with a minimal test case
    • Searched documentation for error message
    • Checked for common issues in this guide

    Quick Reference¶

    Common Fixes¶

    Problem Quick Fix
    Connection refused Check host/port/firewall
    Authentication failed Verify credentials/kinit
    Table exists Enable overwrite or use new name
    Out of memory Increase -Xmx or split job
    Too many partitions Increase limit or use EXPORT
    ACID errors Enable downgrade or use EXPORT
    Timeout Increase timeout or use different strategy
    Permission denied Check user permissions

    Useful Commands¶

    # Check HMS-Mirror is running
curl http://localhost:8090/hms-mirror/api/v1/health

# Test HiveServer2 connectivity
beeline -u "jdbc:hive2://host:10000/default"

# Check Kerberos ticket
klist

# Get new Kerberos ticket
kinit -kt keytab principal@REALM

# Check disk space
df -h ~/.hms-mirror

# View recent logs
tail -100 ~/.hms-mirror/logs/hms-mirror.log

# Find errors in logs
grep -i error ~/.hms-mirror/logs/hms-mirror.log
