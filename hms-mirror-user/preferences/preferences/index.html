    Preferences¶

    Configure HMS-Mirror behavior, security settings, and runtime options through the Preferences section. These settings affect how HMS-Mirror connects to clusters, processes migrations, and integrates with external services.

    Accessing Preferences¶

    Navigate to Settings in the HMS-Mirror web UI to access preference categories.

    Preference Categories¶

    Security¶

    Configure authentication and secure communications:

    Kerberos¶

    Enterprise authentication setup: - Configuring krb5.conf paths - Keytab file management - Principal configuration - Ticket renewal and validation - Troubleshooting Kerberos issues

    Certificates¶

    SSL/TLS certificate management: - Managing truststore and keystore - Importing cluster certificates - Self-signed certificate handling - Certificate validation settings

    Runtime Configurations¶

    Application behavior and performance settings:

    JVM Settings¶

    Java Virtual Machine configuration: - Memory allocation (heap size) - Garbage collection tuning - Performance optimization flags - Debugging options

    LLM Integration¶

    AI-powered analysis features: - Supported LLM providers (Ollama, OpenAI, Anthropic, etc.) - Embedded vs. external LLM setup - Model selection and configuration - Privacy considerations - Enabling/disabling AI features

    Quick Settings Reference¶

    Setting Location Purpose
    Kerberos principal Security → Kerberos Authentication identity
    Truststore path Security → Certificates SSL certificate validation
    JVM heap size Runtime → JVM Memory allocation
    LLM provider Runtime → LLM AI analysis backend

    Common Configuration Tasks¶

    Enable Kerberos Authentication¶

    1. Navigate to Security → Kerberos
    2. Configure krb5.conf location
    3. Set principal and keytab path
    4. Test authentication with klist

    Add Cluster Certificates¶

    1. Navigate to Security → Certificates
    2. Import certificates from clusters
    3. Build combined truststore
    4. Configure connections to use truststore

    Configure LLM Analysis¶

    1. Navigate to Runtime → LLM
    2. Choose provider (embedded Ollama recommended for privacy)
    3. Select model appropriate for your needs
    4. Enable AI features in the UI

    Increase Memory for Large Migrations¶

    1. Navigate to Runtime → JVM
    2. Increase -Xmx setting (e.g., -Xmx8g)
    3. Restart HMS-Mirror server
    4. Verify memory allocation in status page

    Environment-Specific Settings¶

    Development Environment¶

    - Kerberos: May be disabled
- SSL: Self-signed certificates OK
- LLM: Local Ollama or disabled
- Memory: Default (2-4GB)

    Production Environment¶

    - Kerberos: Fully configured
- SSL: Proper CA certificates
- LLM: Per security policy
- Memory: Increased (8GB+)

    Settings Persistence¶

    All preferences are stored in HMS-Mirror's RocksDB database at:

    ~/.hms-mirror/data/

    Settings persist across server restarts. To reset to defaults, clear the RocksDB directory (backup first!).

    Exporting and Importing Settings¶

    Individual configurations can be exported for backup or sharing:

    1. Navigate to the relevant settings page
    2. Click Export to download as JSON/YAML
    3. Use Import to restore on another instance
