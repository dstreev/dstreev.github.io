    Master Password¶

    The Master Password protects the PKCS12 keystore that contains all encryption keys used by HMS-Mirror. Changing the master password re-encrypts the keystore with the new password while preserving all stored encryption keys.

    Note

    The Master Password tab is only visible to users with the ADMIN role and is hidden when Cloudera Manager mode is enabled.

    Overview¶

    HMS-Mirror stores encryption keys in a PKCS12 keystore file at ~/.hms-mirror/keystore/hms-mirror-keys.p12. These keys encrypt all sensitive data including:

    • Database connection passwords
    • Kerberos keytab files
    • Kerberos credential passwords
    • Trino connection passwords
    • HCFS configuration files
    • Cloudera Manager connection passwords

    The master password controls access to this keystore. Without the correct password, HMS-Mirror cannot decrypt any stored credentials.

    For detailed information on the encryption architecture, see the Security Architecture overview.

    Viewing Keystore Status¶

    Navigate to Preferences > Master Password. The status panel displays:

    Field Description
    Status Whether the keystore file exists on disk (Active or Not Found)
    Configured Whether a keystore password has been set
    Keystore Path File system location of the PKCS12 keystore (e.g., ~/.hms-mirror/security/hms-mirror-keys.p12)

    Providing the Password¶

    The master password is resolved in the following order of precedence:

    Option 1: Environment Variable (Recommended)¶

    export HMS_MIRROR_KEYSTORE_PASSWORD="your-secure-password"
java -jar hmsm-server-*.jar

    Option 2: System Property¶

    java -Dhms-mirror.keystore.password="your-secure-password" -jar hmsm-server-*.jar

    Option 3: Default Development Password¶

    If no environment variable or system property is set, HMS-Mirror uses the default development password defined in application.yaml (hms-mirror-dev-keystore-password). This is convenient for development but must be overridden for production deployments since it is a well-known value.

    Changing the Master Password¶

    There are two common scenarios for changing the master password:

    • Initial production setup — replacing the default development password with a secure production password
    • Password rotation — periodically changing an existing production password per your organization's security policy

    Both follow the same process in the UI.

    Step-by-Step Process¶

    1. Navigate to Preferences > Master Password
    2. Review the Encryption Keystore Status panel to confirm the keystore is active
    3. In the Change Master Password form:
      • Enter the Current Password — this is the password HMS-Mirror is currently using (the default hms-mirror-dev-keystore-password if you haven't changed it, or your current production password)
      • Enter the New Password (minimum 8 characters)
      • Enter the new password again in Confirm New Password
    4. Click Change Master Password
    5. On success, a Restart Required alert appears (see below)

    Tip

    Each password field has a visibility toggle (eye icon) so you can verify what you've typed.

    Validation Rules¶

    The form validates the following before submitting:

    Rule Description
    Current password required Cannot be empty
    Current password correct Must match the password the keystore was encrypted with
    New password minimum length At least 8 characters
    Passwords must match New password and confirmation must be identical
    Password must change New password must differ from the current password

    What Happens During the Change¶

    When you click Change Master Password, the server performs these steps:

    1. Validates the current password by loading the existing PKCS12 keystore
    2. Creates a backup of the current keystore with a timestamp suffix (e.g., hms-mirror-keys_20250210_143022.p12) in the same directory
    3. Creates a new keystore encrypted with the new password
    4. Transfers all encryption keys from the old keystore into the new one
    5. Saves the new keystore file and applies restrictive file permissions (0400)
    6. Updates the in-memory password reference for the remainder of the current session

    All encryption keys remain unchanged — only the keystore's password protection is updated. Existing encrypted data (connection passwords, keytabs, etc.) is not affected.

    After Changing: Restart Required¶

    After a successful password change, a prominent Restart Required alert appears with instructions:

    1. Before stopping the application, update your password configuration:

      If using an environment variable: 

      export HMS_MIRROR_KEYSTORE_PASSWORD="your-new-password"

      If using a system property: 

      # Add to your startup command:
-Dhms-mirror.keystore.password="your-new-password"

    2. Restart HMS-Mirror for the new password to take effect on the next startup

    Warning

    If HMS-Mirror is restarted without the correct password configured, it will not be able to open the keystore or decrypt any stored credentials. The application will fail to start properly.

    Note

    The current running session continues to work after the password change because the server updates its in-memory reference. The restart is required so that the new password is loaded from the environment on next startup.

    Complete Walkthrough: Moving to Production¶

    This example shows the full process of replacing the default development password with a production password:

    1. Start HMS-Mirror with the default password (no env var set):

    java -jar hmsm-server-*.jar
# Uses default: hms-mirror-dev-keystore-password

    2. Change the password in the UI:

    • Navigate to Preferences > Master Password
    • Current Password: hms-mirror-dev-keystore-password
    • New Password: My$ecureProductionP@ss2025
    • Confirm: My$ecureProductionP@ss2025
    • Click Change Master Password

    3. Configure the new password in your environment:

    # Add to your deployment scripts, systemd service file, or secrets manager:
export HMS_MIRROR_KEYSTORE_PASSWORD="My$ecureProductionP@ss2025"

    4. Restart HMS-Mirror:

    # Stop the running instance, then restart with the new password available
java -jar hmsm-server-*.jar

    5. Verify — Log in and confirm connections and credentials are accessible.

    Keystore Backups¶

    Each password change creates a timestamped backup in the keystore directory:

    ~/.hms-mirror/security/
  hms-mirror-keys.p12                       # Current keystore
  hms-mirror-keys_20250115_143022.p12       # Backup from Jan 15
  hms-mirror-keys_20250210_091500.p12       # Backup from Feb 10

    Backups are protected with the previous password (the one that was active when the backup was created). These backups allow recovery if the new password is lost — you can restore a backup file over the current keystore and restart with the corresponding password.

    Tip

    Periodically clean up old backup files once you've confirmed the new password is working. Each backup contains the same encryption keys, just protected by a different password.

    Recovery¶

    If you lose the current master password:

    1. Check for backups in ~/.hms-mirror/security/ (or ~/.hms-mirror/keystore/ in older versions)
    2. If you have a backup and remember that backup's password:
      • Stop HMS-Mirror
      • Copy the backup over the current keystore: cp hms-mirror-keys_20250115_143022.p12 hms-mirror-keys.p12
      • Set the environment variable to the backup's password
      • Start HMS-Mirror
    3. If no backup or previous password is available:
      • Delete the keystore file
      • Restart HMS-Mirror (a new keystore will be created)
      • Re-enter all stored credentials (connection passwords, keytabs, etc.)

    Warning

    Deleting the keystore file (hms-mirror-keys.p12) permanently destroys all encryption keys. Any data encrypted with those keys — passwords, keytabs, configuration files — becomes unrecoverable. You will need to reconfigure all connections and re-upload all keytabs.

    Security Considerations¶

    • Override the default — The default development password (hms-mirror-dev-keystore-password) is published in the source code. Always set HMS_MIRROR_KEYSTORE_PASSWORD for any non-development deployment.
    • Use a strong password — Minimum 8 characters with mixed case, numbers, and symbols
    • Use a secrets manager — Store the password in your organization's secrets management solution (HashiCorp Vault, AWS Secrets Manager, etc.) rather than in plain-text scripts
    • File permissions — The keystore file is automatically set to owner-read-only (0400) and the directory to owner-only (0700)
    • Compliance — The encryption key separation architecture complies with PCI-DSS 3.5.2 and NIST SP 800-57
    • Rotate periodically — Follow your organization's password rotation policy by using the change process documented above
