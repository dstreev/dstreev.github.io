    HMS-Mirror Troubleshooting Guide¶

    Overview¶

    This guide covers common issues encountered during HMS-Mirror migrations and their solutions. Issues are organized by category for quick reference.

    Connection Issues¶

    Unable to Connect to HiveServer2¶

    Symptoms: - "Connection refused" error - "Unable to establish connection" message

    Solutions:

    1. Verify HiveServer2 is running: 

      beeline -u "jdbc:hive2://hostname:10000"

    2. Check connection string format: 

      # Standard
jdbc:hive2://hostname:10000/default

# With Kerberos
jdbc:hive2://hostname:10000/default;principal=hive/_HOST@REALM

# With SSL
jdbc:hive2://hostname:10000/default;ssl=true;sslTrustStore=/path/truststore.jks

# With HTTP transport (common in cloud)
jdbc:hive2://hostname:443/default;transportMode=http;httpPath=/hive2

    3. Test network connectivity: 

      telnet hostname 10000
nc -zv hostname 10000

    4. Check firewall rules: Ensure ports 10000 (binary) or 443 (HTTP) are open

    Kerberos Authentication Failures¶

    Symptoms: - "GSS initiate failed" - "Could not obtain Kerberos principal"

    Solutions:

    1. Verify Kerberos ticket: 

      klist
kinit -kt /path/to/keytab principal@REALM

    2. Check principal in connection string: 

      jdbc:hive2://host:10000/default;principal=hive/hostname@REALM

    3. Verify cross-realm trust (for linked clusters): 

      # krb5.conf should have both realms
[realms]
REALM1.COM = { ... }
REALM2.COM = { ... }

[capaths]
REALM1.COM = {
  REALM2.COM = .
}

    HDFS/Cloud Storage Access Issues¶

    Symptoms: - "Permission denied" on storage paths - "Unknown host" for HDFS nameservice

    Solutions:

    1. Verify HDFS access: 

      hdfs dfs -ls hdfs://nameservice/path

    2. Check HA configuration: Ensure hdfs-site.xml has nameservice config

    3. Verify cloud credentials: 

      # S3
aws s3 ls s3://bucket/

# ADLS
az storage blob list --container-name container

    4. Check Hadoop credential provider for cloud secrets

    Migration Execution Issues¶

    Query Timeout¶

    Symptoms: - "Query timeout exceeded" - Long-running queries not completing

    Solutions:

    1. Increase timeout settings: 

      optimization:
  overrides:
    properties:
      hive.server2.idle.operation.timeout:
        BOTH: "14400000"  # 4 hours
      hive.server2.idle.session.timeout:
        BOTH: "28800000"  # 8 hours

    2. Use HYBRID strategy: Large tables auto-select better approach

    3. Migrate large tables separately: Dedicated jobs with higher limits

    Out of Memory Errors¶

    Symptoms: - "Java heap space" errors - "Container killed by YARN for exceeding memory limits"

    Solutions:

    1. Increase container memory: 

      optimization:
  overrides:
    properties:
      hive.tez.container.size:
        BOTH: "16384"
      hive.tez.java.opts:
        BOTH: "-Xmx13107m"

    2. Reduce parallelism: 

      optimization:
  overrides:
    properties:
      hive.exec.parallel.thread.number:
        BOTH: "4"

    3. Process fewer tables concurrently

    Too Many Partitions¶

    Symptoms: - "Exceeded partition limit" - Tables being skipped

    Solutions:

    1. Increase partition limits: 

      hybrid:
  sqlPartitionLimit: 1000  # Increase from 500
  exportImportPartitionLimit: 200  # Increase from 100

    2. Migrate partition ranges: Split into multiple jobs by partition values

    3. Use SCHEMA_ONLY + distcp: For very partition-heavy tables

    ACID Table Issues¶

    Symptoms: - "Cannot read ACID table" - "Transaction timeout"

    Solutions:

    1. Run compaction first: 

      ALTER TABLE db.table COMPACT 'major' AND WAIT;

    2. Enable ACID migration: 

      migrateACID:
  on: true
  downgrade: true

    3. Check for open transactions: 

      SHOW TRANSACTIONS;
SHOW LOCKS DATABASE db;

    Schema/DDL Issues¶

    Table Already Exists¶

    Symptoms: - "Table already exists" error

    Solutions:

    1. Enable createIfNotExists: 

      createIfNotExists: true

    2. Drop existing tables first (if re-running)

    3. Use different target database for testing

    Column Type Mismatch¶

    Symptoms: - "Type mismatch" errors - "Cannot cast" errors

    Solutions:

    1. Review type mappings: Some types differ between Hive versions

    2. Use SQL strategy with CAST: Explicit type conversion

    3. Modify source schema: If feasible, align types

    View Dependencies¶

    Symptoms: - "View references non-existent table"

    Solutions:

    1. Ensure base tables migrate first: Views are processed last

    2. Check cross-database references: Views may reference tables in other databases

    3. Migrate dependent databases together

    Performance Issues¶

    Slow Migration¶

    Symptoms: - Migration taking much longer than expected

    Solutions:

    1. Enable auto-tuning: 

      optimization:
  autoTune: true

    2. Increase parallelism: 

      optimization:
  overrides:
    properties:
      hive.exec.parallel:
        BOTH: "true"
      hive.exec.parallel.thread.number:
        BOTH: "8"

    3. Use dedicated queue: 

      optimization:
  overrides:
    properties:
      tez.queue.name:
        BOTH: "migration"

    4. Schedule during off-peak hours

    Data Skew¶

    Symptoms: - Some partitions/tables much slower than others - Uneven resource utilization

    Solutions:

    1. Use DISTRIBUTE BY (HMS-Mirror adds automatically)

    2. Split skewed partitions: Migrate separately

    3. Increase reducer count for skewed tables

    Validation Issues¶

    Row Count Mismatch¶

    Symptoms: - Source and target counts don't match

    Solutions:

    1. Check for in-flight transactions: Wait for completion

    2. Verify partition coverage: All partitions migrated?

    3. Check for filtered data: Review dataset filters

    4. Re-run for specific tables: With fresh source state

    Missing Partitions¶

    Symptoms: - Partitions exist on source but not target

    Solutions:

    1. Run MSCK REPAIR TABLE: 

      MSCK REPAIR TABLE db.table;

    2. Verify data was copied: Check storage locations

    3. Check partition limits: May have been skipped

    Web UI Issues¶

    Cannot Access HMS-Mirror UI¶

    Symptoms: - Browser shows connection error - 404 or 500 errors

    Solutions:

    1. Verify server is running: 

      ps aux | grep hms-mirror
curl http://localhost:8080/hms-mirror/health

    2. Check port availability: 

      netstat -tlnp | grep 8080

    3. Review server logs: Check for startup errors

    Connection Test Failing¶

    Symptoms: - Test connection shows "Failed" in UI

    Solutions:

    1. Use Diagnose feature: Click "Diagnose" for AI-powered analysis

    2. Check driver availability: Ensure JDBC drivers are installed

    3. Verify network from server: HMS-Mirror server needs access, not just your browser

    Recovery Procedures¶

    Recovering from Failed Migration¶

    1. Identify failure point: Check execution logs

    2. Clean up partial data: Drop incomplete tables if needed

    3. Fix configuration: Address root cause

    4. Re-run failed tables only: Create dataset with just failed tables

    5. Validate results: Compare with source

    Rolling Back Migration¶

    1. Drop migrated tables on target: 

      DROP TABLE db.table;

    2. Verify source intact: Confirm source wasn't modified

    3. Analyze failure: Determine what went wrong

    4. Re-plan migration: Adjust approach as needed

    Getting Help¶

    Information to Gather¶

    When seeking help, collect:

    1. HMS-Mirror version: Check About page or startup logs
    2. Error messages: Exact error text
    3. Configuration: Relevant settings (sanitized)
    4. Execution logs: Full log output
    5. Environment: Platform types, versions

    Using Diagnose Feature¶

    For connection errors: 1. Click "Test" on connection 2. When test fails, click "Diagnose" 3. AI provides analysis and suggestions 4. Follow recommended steps

    Log Locations¶

    • Server logs: logs/hms-mirror.log
    • Execution reports: reports/<job-id>/
    • Console output: stdout during execution

    Common Error Messages¶

    Error Likely Cause Solution
    "Connection refused" HS2 not running Start HiveServer2
    "GSS initiate failed" Kerberos issue Renew ticket, check principal
    "Permission denied" Access rights Check ACLs, IAM roles
    "Table already exists" Re-running job Enable createIfNotExists
    "Query timeout" Large table Increase timeout or use different strategy
    "Container killed" Memory exhaustion Increase container memory
    "Too many partitions" Partition limit Increase limit or split migration
    "Unknown host" DNS/network Check nameservice config
