    Release Notes¶

    This page documents the changes, improvements, and fixes in each release of HMS-Mirror. For upgrade instructions and compatibility information, refer to the specific version notes below.

    4.1.5.10¶

    Release Date: March 2026 Previous Version: 4.1.5.0

    Highlights¶

    This is a major feature release that introduces AI Agent integration with Claude Code CLI, a Cloudera Platform Framework (CPF) plugin for embedding HMS-Mirror inside Cloudera Manager, MCP (Model Context Protocol) security with per-user JWT authentication, a comprehensive CMA-aligned UI redesign, security hardening from a full threat model review, and Docker support for Claude CLI.

    New Features¶

    AI Agent Integration (Claude Code CLI)¶

    • Claude Agent Sessions — New AgentSessionManager and AgentSession provide synchronous Claude CLI interaction for per-user AI analysis
    • Claude CLI Manager — Shared ClaudeCliManager handles CLI detection, context directory deployment, MCP config creation, and API key management
    • AI Analysis for Configs, Jobs, Reports, and Connections — One-click AI analysis across all major entity pages using the agent-based analysis pattern
    • Analysis Tracker — Persistent analysisTracker service preserves AI feedback across page navigation
    • Analysis Banner — Reusable AnalysisBanner component displays AI analysis results inline on any page
    • Chat Session Export — Export Assistant conversations to Markdown with the download button on chat tabs
    • Session Expiry Handling — Chat sessions show timeout banners and disable input for expired sessions
    • Feature Flag Consolidation — Unified hms-mirror.features.llm.enabled flag replaces separate assistant.enabled and llm.enabled flags

    CPF Plugin (Cloudera Platform Framework)¶

    • PF4J Plugin — HMS-Mirror runs as a headless plugin inside Cloudera Manager Application (CMA), sharing CMA's authentication and navigation
    • DispatcherServlet Router — Lightweight (~180 lines) Spring MVC router replaces the 1,482-line custom request router for dispatching REST calls within the plugin classloader
    • MCP Tool Provider — Plugin implements McpToolProvider to expose HMS-Mirror tools through CPF's native MCP integration
    • Plugin-Mode UI Filtering — Frontend adapts layout (no header, sidebar adjustments) when running inside CMA
    • CM Proxy Client — Plugin routes Cloudera Manager API calls through CPF's built-in proxy instead of direct HTTP

    MCP Security¶

    • Per-User JWT Authentication — MCP endpoints authenticated via per-conversation JWT tokens with 8-hour TTL
    • MCP Authentication Filter — Dedicated McpAuthenticationFilter validates Bearer tokens on /mcp endpoints
    • WebSocket Authentication — JWT-based WebSocketAuthInterceptor secures Socket.IO connections
    • API Key Management — Self-service API key creation and management via Profile page with RocksDB storage
    • Forwarding Credentials — JwtTokenProvider generates forwarding tokens for MCP session paths, ensuring tool calls run with the originating user's identity

    Security Hardening (Threat Model)¶

    • Account Lockout Protection (T08) — Rate-limiting LoginRateLimitFilter with configurable thresholds, lazy auto-unlock after cooldown, and admin unlock endpoint
    • Swagger UI Gating (T07) — Swagger UI disabled by default; when enabled, requires ROLE_ADMIN
    • Admin Account Unlock UI — Lock status badges, unlock buttons, and unlock dialog in the user management page
    • Configurable CORS Origins (T09) — HMS_MIRROR_CORS_ALLOWED_ORIGINS environment variable and cors.allowed.origins property for production deployments
    • User Registration Toggle — HMS_MIRROR_REGISTRATION_ENABLED controls whether open self-registration is allowed

    Docker Claude CLI Support¶

    • Node.js 20 in Docker Image — Rocky Linux 9 Docker image now includes Node.js 20 LTS for Claude CLI
    • Claude CLI Pre-installed — @anthropic-ai/claude-code npm package installed during Docker build
    • ANTHROPIC_API_KEY — New environment variable passthrough in Dockerfile and docker-compose.yml

    Improvements¶

    UI/UX — CMA Visual Alignment¶

    • CMA Design System — Complete visual alignment with Cloudera Manager Application design tokens (colors, typography, spacing, border-radius)
    • Roboto Font — Typography updated to match CMA's Roboto font family
    • CUIX Icons — Replaced Heroicons with Cloudera CUIX icon set across all components
    • CMA Layout Components — New PageHeader, PageSubtitleBar, Breadcrumb, DashboardPanel, and FormField components matching CMA patterns
    • Dark Sidebar — Permanently dark sidebar matching CMA navigation style
    • Page Migrations — All pages (6 batches) migrated to CMA layout components

    JWT Secret Management¶

    • JWT Secret UI — Admin interface for viewing, generating, and persisting JWT secrets under Preferences > Security
    • Startup Script Support — HMS_MIRROR_JWT_SECRET environment variable and jwt.secret property in web.properties
    • Graceful Fallback — Random JWT secret generated on startup if none configured (sessions lost on restart)

    CSD and Parcel¶

    • CSD Sync — control.sh synced with latest application changes, JWT secret parameter added
    • TLS Default — Default TLS protocol updated to TLSv1.3

    Build System¶

    • Skip Frontend/MkDocs Builds — Maven properties skip.frontend.build and skip.mkdocs.build for faster iteration
    • Decoupled CMA Plugin POM — Plugin POM decoupled from reactor parent to avoid CMA dependency conflicts

    Test Infrastructure¶

    • E2E Test Framework — New v4 test case framework with ConversionResultTestFactory, migrated legacy-to-CDP test cases
    • RocksDB Repository Fix — Fixed double key-prefix bug in AbstractRocksDBRepository.deleteById() that caused incorrect return values

    Bug Fixes¶

    • Fixed stripDatabase failing on backtick-quoted `db`.`table` format
    • Fixed database SQL not included in schema-only migration response
    • Fixed setDbRename for dbSpec — migrating tables from source to differently-named target database now works correctly (AMCM-3941)
    • Fixed Assistant chat sessions not persisting across page navigation
    • Fixed main layout dynamic left nav display
    • Fixed connection copy to correctly handle all fields
    • Fixed ZooKeeper-imported connections not picking up SSL configurations
    • Fixed PasswordEncoder interface usage instead of BCryptPasswordEncoder concrete type
    • CVE dependency updates for Netty and hcfs-browser

    Upgrade Notes¶

    1. Feature Flag Change — hms-mirror.features.assistant.enabled has been removed. Use hms-mirror.features.llm.enabled to control all AI/LLM features.
    2. JWT Secret — For production deployments, configure HMS_MIRROR_JWT_SECRET or jwt.secret in web.properties to persist sessions across restarts.
    3. Docker — The Docker image now includes Node.js 20 and Claude CLI. Pass ANTHROPIC_API_KEY to enable AI features.
    4. CORS — Production deployments behind a reverse proxy should set HMS_MIRROR_CORS_ALLOWED_ORIGINS to the actual origin URL.
    5. Swagger — Swagger UI is now disabled by default. Enable with springdoc.swagger-ui.enabled=true (requires ADMIN role).

    4.1.5.0¶

    Release Date: February 2026 Previous Version: 4.0.0.6

    Highlights¶

    This is a major release that introduces Cloudera Manager integration for automated cluster discovery and configuration import, enterprise SSO via OIDC authentication, LDAP directory authentication, a PKCS12 keystore for PCI-DSS/NIST-compliant encryption key management, a new Job Build Wheel UI, Cloudera Manager Parcel packaging, and a complete documentation overhaul using MkDocs.

    New Features¶

    Cloudera Manager Integration¶

    • CM Connector — Discover and import Hadoop/Hive configurations directly from Cloudera Manager instances
    • Automated HCFS Profile Import — Import HDFS/OFS/S3 storage configurations with namespace and warehouse settings
    • Automated HS2 Connection Import — Import HiveServer2 endpoints with JDBC URIs, ZooKeeper discovery, and SSL/TLS settings
    • Kerberos Detection — Automatic detection of Kerberos requirements during CM import with credential assignment
    • CM Connection Management — Create, test, and manage multiple Cloudera Manager connections with encrypted credential storage
    • CM Feature Flag — hms-mirror.features.cm.enabled flag controls visibility of CM-specific UI features and hides non-applicable preferences (JVM, TLS, Authentication) in CM mode

    OIDC Authentication (OAuth2 / OpenID Connect)¶

    • Single Sign-On — Connect HMS-Mirror to identity providers for enterprise SSO authentication
    • Provider Presets — Pre-configured issuer URIs for Google, Okta, Azure AD, and Keycloak
    • Group-Based Authorization — Map OIDC groups to HMS-Mirror roles (ADMIN / USER)
    • Dynamic User Provisioning — Auto-create local accounts on first OIDC login
    • OIDC Configuration UI — Full configuration interface under Preferences > Authentication

    LDAP Authentication¶

    • Enterprise Directory Integration — Connect to Active Directory, FreeIPA, or OpenLDAP for centralized authentication
    • Quick Setup Presets — Auto-fill common directory configurations for AD, FreeIPA, and OpenLDAP
    • LDAP Connection Testing — Test LDAP connectivity, bind credentials, and user search configuration from the UI
    • Group-Based Authorization — Map LDAP groups to HMS-Mirror user and admin roles
    • SSL/STARTTLS Support — Secure LDAP connections with SSL (LDAPS) or STARTTLS

    Master Password & Keystore Security¶

    • PKCS12 Keystore — Encryption keys stored in a password-protected PKCS12 keystore, separate from encrypted data in RocksDB
    • Key Separation Compliance — Meets PCI-DSS 3.5.2 and NIST SP 800-57 requirements for cryptographic key separation
    • Master Password UI — Admin-only interface for changing the keystore password with backup creation
    • External Password Configuration — Support via HMS_MIRROR_KEYSTORE_PASSWORD environment variable or system property
    • Automatic Key Migration — Existing encryption keys in RocksDB automatically migrate to the keystore on first startup
    • CM Connection Encryption — New cm_connections_key domain for encrypting Cloudera Manager credentials

    User Management¶

    • Admin User Management — Administrators can reset passwords, grant/revoke admin roles, and enable/disable user accounts
    • Self-Service Profile — All users can update their email and change their password
    • Role-Based Access — ADMIN and USER roles with appropriate feature access controls

    Job Build Wheel Interface¶

    • Circular Wheel UI — New visual interface with 5 segments (Job Details, Data Strategy, References, Options, Summary)
    • Non-Linear Navigation — Click any segment to jump directly to that section
    • Visual Progress Tracking — Segments show completion status and error indicators
    • Animated Segment Cards — Smooth transitions with Framer Motion animations
    • Central Hub Display — Shows job name and overall completion percentage
    • User Preference Toggle — Choose between Wheel and Wizard interfaces via Settings

    TLS/HTTPS Configuration¶

    • Self-Signed Certificate Generation — Generate server certificates directly from the Preferences UI
    • Configurable Certificate Fields — CN, hostname, organization, validity, key size, TLS protocol, SANs
    • Certificate Status Monitoring — View generation date, expiry, and color-coded validity indicators
    • TLS Enable/Disable — Toggle HTTPS access with a restart

    Cloudera Manager Parcel Packaging¶

    • Parcel Build — Automated Maven build of CM-compatible parcel (.parcel file with SHA checksum)
    • CSD (Custom Service Descriptor) — Service descriptor for CM integration
    • Multi-Platform Support — Build profiles for EL7, EL8, EL9, and Ubuntu

    MkDocs Documentation¶

    • Documentation Overhaul — Migrated from JetBrains Writerside to MkDocs with Material theme
    • Integrated Documentation — Docs served directly from the application at /docs/
    • In-App Help Links — Context-sensitive help icons throughout the UI linking to relevant documentation pages
    • Search — Full-text search across all documentation pages
    • Dark/Light Mode — Documentation theme matches user preference

    Improvements¶

    Application Preferences¶

    • Dataset Page Preference — Choose between Builder (drag-and-drop) and Wizard (step-by-step) for dataset creation
    • Job Page Preference — Choose between Wheel (circular) and Wizard (step-by-step) for job creation
    • Persistent Preferences — User preferences stored in RocksDB across sessions

    Job Management¶

    • Job Copy — Copy existing jobs as templates for new ones
    • Job Validation — Pre-run validation with error and warning dialogs
    • Dry-Run Support — Validate job configuration without executing changes
    • Grid and List Views — Toggle between display modes for job lists
    • Advanced Filtering — Filter jobs by strategy, dataset, configuration, and connections
    • Section-Based Form — Job form refactored into independent section components with shared state via useJobForm hook

    Connection Management¶

    • ZooKeeper SSL Fix — Imported connections from Cloudera Manager now correctly pick up SSL configurations for ZooKeeper-based HS2 discovery
    • HCFS Profile Support — Intermediate storage and target namespace selection via HCFS profiles

    Keystore File Permissions¶

    • Restrictive POSIX Permissions — Keystore file set to 0400 (owner read-only), directory set to 0700 (owner only)
    • Automatic Backup — Timestamped backup created before each password change

    Upgrade Notes¶

    1. Java 17+ Required — Minimum Java version remains Java 17
    2. Master Password — The default development password should be overridden for production. See Master Password documentation.
    3. Automatic Key Migration — On first startup after upgrade, encryption keys are automatically migrated from RocksDB to the PKCS12 keystore. No action required.
    4. Authentication — If switching from Local to LDAP or OIDC authentication, configure under Preferences > Authentication and restart. See Authentication.
    5. Documentation — Documentation is now served from /docs/ within the application. The Writerside documentation has been replaced.
    6. Parcel Deployment — For Cloudera Manager deployments, use the parcel build: mvn clean package -pl parcel -am

    4.0.0.6¶

    Release Date: December 9, 2025 Previous Version: 4.0.0.5 Commits: 4 commits, 41 files changed (+4,589 / -178 lines)

    Highlights¶

    This release introduces the Dataset Builder with HCFS directory browsing, Application Preferences for UI customization, CLI Integration feature flags, and fixes for RocksDB backup functionality.

    New Features¶

    Dataset Builder¶

    • Interactive Dataset Builder Page - New drag-and-drop interface for building datasets with visual database and table selection
    • HCFS Directory Browser Dialog - Browse and select warehouse directories directly from HCFS (HDFS/OFS/S3) storage within the Dataset Builder
    • Create Directory Support - Ability to create new directories directly from the HCFS browser dialog
    • Dynamic Schema Tooltips - Tooltips now pull descriptions dynamically from @Schema annotations for accurate field documentation
    • Directory Validation - Warehouse directory fields validate that paths don't contain database names (automatically appended during location translations)

    Application Preferences¶

    • Dataset Editor Preference - New Settings page under Utilities to choose between Builder (drag-and-drop) and Wizard (step-by-step) interfaces for dataset creation
    • Persistent Preferences - Application preferences stored in RocksDB for persistence across sessions

    Improvements¶

    RocksDB Backup System¶

    • Fixed Backup Persistence - Resolved issue where backups would disappear after creation due to incorrect BackupEngine usage
    • Proper BackupEngine Usage - Now uses single persistent backup directory with RocksDB managing multiple backups internally via backup IDs
    • Backup Names with Timestamps - Backup names now include timestamps (e.g., backup_1_2025-12-09_14-30-45) for better organization
    • Enhanced Backup Logging - Added detailed logging for backup initialization, execution, and completion
    • Manual Backup Trigger API - New POST /api/v1/preferences/backup/trigger endpoint for testing backup configuration

    Dataset Management¶

    • Fixed Edit Mode - Dataset Builder now properly calls update API when editing existing datasets (was incorrectly calling create)
    • Connection Copy Fix - Fixed bug when copying connections

    Bug Fixes¶

    • Fixed Connection Copy functionality not working correctly
    • Fixed Dataset edit mode throwing "Dataset exists already" error
    • Fixed RocksDB backups disappearing immediately after creation
    • Fixed backup scheduler unnecessarily rescheduling after each execution
