    Command-Line Reference Guide¶

    This guide covers installing and running HMS-Mirror components from the command line, including the Web Server, CLI, and Client modules.

    Prerequisites¶

    Java Requirements¶

    HMS-Mirror requires Java 17 or higher. The installation scripts will verify your Java version.

    # Check your Java version
java -version

# Set JAVA_HOME if needed
export JAVA_HOME=/path/to/java17

    Network Requirements¶

    Ensure network access to your DataLake environments: - HiveServer2: Port 10000 (binary) or 10001 (HTTP) - Metastore: Port 9083 (if using direct access) - Knox Gateway: Port 8443 (if applicable)

    Installation¶

    Download the Distribution¶

    Download the HMS-Mirror distribution archive and extract it:

    # Extract the server distribution
tar -xzf hmsm-server-<version>.tar.gz

# Or extract the CLI distribution
tar -xzf hmsm-cli-<version>.tar.gz

    User Installation (Recommended)¶

    User installation places files in ~/.hms-mirror/ and creates symlinks in ~/bin/. This is the recommended approach for most users.

    Server Installation¶

    cd hmsm-server-<version>
./bin/setup.sh

    CLI Installation¶

    cd hmsm-cli-<version>
./bin/setup.sh

    Installation locations (non-root):

    Component Location
    Binaries ~/.hms-mirror/bin/
    Libraries ~/.hms-mirror/lib/
    Configurations(Legacy-CLI) ~/.hms-mirror/cfg/
    JDBC Drivers ~/.hms-mirror/drivers/
    Auxiliary JARs ~/.hms-mirror/aux_libs/
    Settings ~/.hms-mirror/settings/
    Reports (Legacy-CLI) ~/.hms-mirror/reports/
    RocksDB Data ~/.hms-mirror/data/
    Logs ~/.hms-mirror/logs/

    Add to PATH:

    After this installation, symlinks are created in ~/bin/ for user installations to run the server and CLI. 

    # Add to ~/.bashrc or ~/.zshrc
export PATH="$HOME/bin:$PATH"

    Root Installation (System-wide)¶

    Running the setup script as root installs to /usr/local/hms-mirror/ and creates symlinks in /usr/local/bin/.

    sudo ./bin/setup.sh

    Installation locations (root):

    Component Location
    Binaries /usr/local/hms-mirror/bin/
    Libraries /usr/local/hms-mirror/lib/
    Symlinks /usr/local/bin/hmsm-server, /usr/local/bin/hmsm-cli
    User Data ~/.hms-mirror/ (per-user)

    Note

    Even with root installation, user-specific data (configurations, drivers, reports) is stored in each user's ~/.hms-mirror/ directory.

    Directory Structure¶

    After installation, the following directory structure is created:

    ~/.hms-mirror/
├── aux_libs/          # Additional JARs for classpath (e.g., cloud storage libs)
├── cfg/               # Configuration files (YAML configs)
├── drivers/           # JDBC drivers for Hive and metastore databases
│   ├── apache/hive/   # Apache Hive drivers by version
│   ├── cdh/           # CDH Hive drivers
│   ├── cdp/           # CDP Hive drivers
│   ├── hdp/           # HDP Hive drivers
│   ├── mysql/         # MySQL metastore drivers
│   ├── postgres/      # PostgreSQL metastore drivers
│   ├── oracle/        # Oracle metastore drivers
│   └── mariadb/       # MariaDB metastore drivers
├── logs/              # Application logs
├── reports/           # Migration reports and output
├── data/              # RocksDB persistent storage (server only)
├── security/          # SSL certificates and truststores
└── settings/          # Properties files
    ├── cli.properties     # CLI JVM and logging settings
    └── web.properties     # Server JVM and logging settings

    JDBC Drivers¶

    HMS-Mirror requires JDBC drivers to connect to Hive and metastore databases.

    Download Hive Drivers¶

    For the Legacy CLI interface, use the provided script to download Hive JDBC drivers:

    # Download all Hive drivers
~/.hms-mirror/bin/download-hive-drivers.sh

# Download to custom location
~/.hms-mirror/bin/download-hive-drivers.sh -d /custom/path/drivers

# Download specific drivers only
~/.hms-mirror/bin/download-hive-drivers.sh -s cdp7_1_9,cdp7_3_1

    Available Hive driver types:

    Type Platform Hive Version
    apache2 Apache Hive 2.3.10
    apache3 Apache Hive 3.1.3
    apache4 Apache Hive 4.0.1
    apache4_1 Apache Hive 4.1.0
    cdh5 Cloudera CDH 5 1.1.0-cdh5.16.99
    cdh6 Cloudera CDH 6 2.1.1-cdh6.3.4
    hdp2 Hortonworks HDP 2 1.2.1000.2.6.5.0-292
    cdp7_1_9 Cloudera CDP 7.1.9 3.1.3000.7.1.9.1059-4
    cdp7_3_1 Cloudera CDP 7.3.1 3.1.3000.7.3.1.404-1

    Download Metastore Drivers¶

    Download JDBC drivers for metastore database connections:

    # Download all metastore drivers
~/.hms-mirror/bin/download-metastore-drivers.sh

# Download specific drivers
~/.hms-mirror/bin/download-metastore-drivers.sh -s mysql,postgres

# Download with custom version
~/.hms-mirror/bin/download-metastore-drivers.sh -s mysql -v mysql:8.0.33

    Available metastore driver types:

    Type Driver Default Version
    mysql MySQL Connector/J 8.2.0
    postgres PostgreSQL JDBC 42.7.2
    oracle Oracle ojdbc11 23.9.0.25.07
    mariadb MariaDB Connector/J 3.5.6

    Running the Server¶

    The HMS-Mirror web server provides the web UI and REST API.

    Starting the Server¶

    # Start the server (runs in background)
hmsm-server

# Start with custom port
hmsm-server --server.port=8091

# Start with custom output directory
hmsm-server -o /path/to/reports

    Stopping the Server¶

    # Stop the running server
hmsm-server --stop

    Server Command-Line Options¶

    Option Description
    --server.port=PORT Set the HTTP port (default: 8090)
    -o, --output-dir DIR Set the report output directory
    --aux-libs DIR Set auxiliary libraries directory
    --show-cp Display the classpath
    --stop Stop the running server
    -DKEY=VALUE Pass JVM system properties
    -CP PATH Add to classpath

    Accessing the Web UI¶

    After starting, access the web interface at:

    http://localhost:8090/hms-mirror

    Server JVM Configuration¶

    Manage JVM settings through the Web UI under Preferences / Utilities / JVM. Settings are stored in ~/.hms-mirror/settings/web.properties:

    # JVM Settings
jvm.initial.heap.size=4096m
jvm.max.heap.size=8192m
jvm.garbage.collector=G1GC
jvm.prefer.ipv4.stack=true
jvm.additional.options=

# Logging
logging.threshold=INFO
logging.file.prefix=hmsm-service
logging.dir=

    Note

    When using the Embedded Ollama LLM, the LLM runs as a separate native process and its memory is managed by the operating system, not the JVM.

    Running the CLI (Legacy)¶

    The CLI provides command-line batch migration capabilities.

    Basic Usage¶

    # Run with configuration file
hmsm-cli -cfg /path/to/config.yaml

# Dry run (validate without executing)
hmsm-cli -cfg /path/to/config.yaml --dry-run

# Specify output directory
hmsm-cli -cfg /path/to/config.yaml -o /path/to/reports

    CLI Command-Line Options¶

    Option Description
    -cfg, --config FILE Configuration file path
    -o, --output-dir DIR Report output directory
    --dry-run Validate without executing
    -db, --database DB Database to process
    -tf, --table-filter REGEX Table name filter (regex)
    -edbf, --exclude-database-filter REGEX Exclude databases matching pattern
    -etf, --exclude-table-filter REGEX Exclude tables matching pattern
    --aux-libs DIR Auxiliary libraries directory
    --show-cp Display classpath
    -p, --password PWD Encrypt a password
    -dp, --decrypt-password PWD Decrypt a password
    -DKEY=VALUE Pass JVM system properties

    Password Encryption¶

    HMS-Mirror supports encrypted passwords in configuration files:

    # Encrypt a password
hmsm-cli -p 'mypassword'

# Decrypt a password
hmsm-cli -dp 'encrypted_password_string'

    Use the encrypted output in your configuration file.

    CLI JVM Configuration¶

    JVM settings are controlled via ~/.hms-mirror/settings/cli.properties:

    # JVM Settings
jvm.initial.heap.size=4096m
jvm.max.heap.size=8192m
jvm.garbage.collector=G1GC
jvm.prefer.ipv4.stack=true
jvm.additional.options=

# Logging
logging.threshold=INFO
logging.file.prefix=hmsm-cli
logging.dir=

    CLI Example Configurations¶

    High-memory for large migrations: 

    jvm.initial.heap.size=8g
jvm.max.heap.size=16g
jvm.garbage.collector=G1GC

    Debug mode: 

    logging.threshold=DEBUG
jvm.additional.options=-XX:+HeapDumpOnOutOfMemoryError

    Kerberos: 

    jvm.additional.options=-Djava.security.krb5.conf=/etc/krb5.conf -Dsun.security.krb5.debug=true

    Custom SSL truststore: 

    jvm.additional.options=-Djavax.net.ssl.trustStore=/path/to/truststore.jks -Djavax.net.ssl.trustStorePassword=changeit

    Running the Client¶

    The client module provides a Java library for programmatic access to HMS-Mirror server APIs.

    Including in Your Project¶

    Add the client library to your project:

    Maven: 

    <dependency>
    <groupId>com.cloudera.utils.hms</groupId>
    <artifactId>hms-mirror-client</artifactId>
    <version>4.0.0.x</version>
</dependency>

    Using the Client JAR¶

    The client is distributed as a fat JAR containing all dependencies:

    java -cp hms-mirror-client-<version>.jar com.example.YourClass

    Configuration Files¶

    Configuration File Location for Legacy CLI¶

    HMS-Mirror looks for configuration files in these locations:

    1. Path specified via -cfg option
    2. ~/.hms-mirror/cfg/
    3. Current working directory

    Sample Configuration¶

    Create a YAML configuration file:

    # Sample HMS-Mirror configuration
clusters:
  LEFT:
    hiveServer2:
      uri: "jdbc:hive2://left-hs2.example.com:10000/default"
      driverClassName: "org.apache.hive.jdbc.HiveDriver"
    partitionDiscovery:
      auto: true
      initMSCK: true

  RIGHT:
    hiveServer2:
      uri: "jdbc:hive2://right-hs2.example.com:10000/default"
      driverClassName: "org.apache.hive.jdbc.HiveDriver"
    partitionDiscovery:
      auto: true
      initMSCK: true

transfer:
  warehouse:
    managedDirectory: "/warehouse/tablespace/managed/hive"
    externalDirectory: "/warehouse/tablespace/external/hive"

databases:
  - "my_database"

    Environment Variables¶

    JAVA_HOME¶

    Specifies the Java installation to use:

    export JAVA_HOME=/usr/lib/jvm/java-17-openjdk

    JAVA_OPTS¶

    Overrides all JVM settings from properties files. Note that JVM memory settings and garbage collectors are setting in the above properties files.

    export JAVA_OPTS="xxxx"

    HMS_MIRROR_AUX_LIBS¶

    Specifies the auxiliary libraries directory:

    export HMS_MIRROR_AUX_LIBS=/custom/path/aux_libs

    Troubleshooting¶

    Java Version Error¶

    Error: Java version "1.8.0_xxx" is too old!
Please upgrade to Java 17 or higher

    Solution: Install Java 17+ and set JAVA_HOME or update your PATH.

    Driver Not Found¶

    No suitable driver found for jdbc:hive2://...

    Solution: Download the appropriate Hive driver: 

    ~/.hms-mirror/bin/download-hive-drivers.sh -s cdp7_3_1

    Permission Denied¶

    Permission denied: /usr/local/hms-mirror/...

    Solution: Run as root for system-wide installation, or use user installation (recommended).

    Process Already Running¶

    Process is already running with PID: xxxxx

    Solution: Stop the existing process first: 

    hmsm-server --stop

    Connection Timeout¶

    Solution: Verify network connectivity and increase timeout in connection properties: 

    socketTimeout=120000

    Quick Reference¶

    Server Commands¶

    hmsm-server                          # Start server
hmsm-server --stop                   # Stop server
hmsm-server --server.port=8091       # Custom port
hmsm-server -o /path/to/reports      # Custom output directory

    CLI Commands¶

    hmsm-cli -cfg config.yaml            # Run migration
hmsm-cli -cfg config.yaml --dry-run  # Dry run
hmsm-cli -p 'password'               # Encrypt password

    Driver Download Commands¶

    download-hive-drivers.sh             # All Hive drivers
download-hive-drivers.sh -s cdp7_3_1 # Specific driver
download-metastore-drivers.sh        # All metastore drivers
download-metastore-drivers.sh -s mysql,postgres  # Specific drivers
