    What's New in HMS-Mirror v4¶

    This document highlights the significant new features and improvements in HMS-Mirror v4 compared to the previous Thymeleaf-based web interface and the legacy CLI version.

    Overview¶

    HMS-Mirror v4 represents a complete architectural redesign with a modern React-based single-page application (SPA), persistent storage with RocksDB, enterprise-grade security features, AI-powered analysis via Claude Agent integration, and a Cloudera Platform Framework (CPF) plugin for embedding inside Cloudera Manager. The application now provides a comprehensive web-based interface for managing complex Hive metadata migrations.

    Check out the Release Notes for a detailed list of new features, improvements, and issues fixed in a release.

    1. Modern React-Based Web Interface¶

    Previous Versions¶

    • CLI: Command-line only, required manual configuration file editing
    • Thymeleaf: Server-rendered pages with limited interactivity

    What's New¶

    • Single Page Application (SPA) built with React and Tailwind CSS
    • Responsive Design with navigation sidebar and card-based layouts
    • Real-time UI Updates with Redux state management
    • Modern UX Patterns including wizards, modals, toast notifications, and loading states
    • Context-Aware Navigation that adapts based on loaded configuration

    2. Connection Management¶

    Previous Versions¶

    • CLI: Single configuration file with LEFT/RIGHT cluster definitions
    • Thymeleaf: Basic connection editing through forms

    What's New¶

    • Multi-Connection Support: Manage unlimited named connections
    • Connection Wizard: Multi-step guided setup including:
    • Basic info (name, description, environment type)
    • HiveServer2 configuration with connection properties
    • Metastore Direct connectivity options
    • Connection pooling configuration
    • Warehouse path configuration
    • Test and validation step
    • Real-time Connection Testing: Test individual components (HCFS, HiveServer2, Metastore)
    • Driver Availability Checking: Verify required JDBC drivers before connecting
    • Partition Discovery Configuration: Auto-discovery and MSCK settings per connection
    • Bulk Operations: Test and manage multiple connections simultaneously
    • Legacy Config Import: Import connections from CLI configuration files

    3. Dataset Management¶

    Previous Versions¶

    • CLI: Database/table filtering via command-line arguments or config file
    • Thymeleaf: Limited dataset definition capabilities

    What's New¶

    • Dataset Wizard: Create targeted migration datasets by:
    • Selecting source and target connections
    • Choosing specific databases and tables
    • Configuring database-specific settings
    • Reviewing and saving for reuse
    • Named Datasets: Save and reuse dataset definitions
    • Dataset Listing: Browse, search, and filter datasets
    • Dataset Validation: Verify dataset definitions before use

    4. Job Management and Scheduling¶

    Previous Versions¶

    • CLI: Single execution per invocation, no job persistence
    • Thymeleaf: Basic execution without job management

    What's New¶

    • Persistent Job Definitions: Create, save, and reuse migration jobs
    • Job Definition Wizard: Guided job creation with:
    • Basic info (name, description)
    • Strategy selection (SQL, Distcp, Hybrid, Export/Import, etc.)
    • Configuration and dataset selection
    • Advanced execution parameters
    • Job Validation: Pre-execution validation with error and warning reporting
    • Job Execution Dialog: Run jobs with configurable execution options
    • Job Status Tracking: Monitor running jobs in real-time
    • Job History: Track execution history and results
    • Duplicate Jobs: Clone existing jobs for similar migrations
    • LLM Job Analysis: AI-powered analysis of job configurations

    5. Kerberos Authentication Management¶

    Previous Versions¶

    • CLI: Manual kinit before execution, no integrated management
    • Thymeleaf: No Kerberos management interface

    What's New¶

    • Kerberos Preferences Panel: Full Kerberos configuration UI
    • Keytab Upload: Upload and manage keytab files through the UI
    • Principal Configuration: Configure Kerberos principal settings
    • Automatic Ticket Refresh: Cron-based scheduler for automatic ticket renewal
    • Ticket Status Monitoring: Real-time display of ticket validity
    • Manual Refresh: On-demand ticket refresh capability
    • Auto-Recovery: Automatic re-authentication on ticket expiration

    6. SSL Certificate Management¶

    Previous Versions¶

    • CLI: Manual JVM truststore configuration
    • Thymeleaf: No certificate management

    What's New¶

    • Certificates Page: Centralized SSL certificate management
    • Truststore Import: Upload certificates to application truststore
    • Bulk Import: Import multiple certificates from a truststore file
    • Certificate Listing: View all certificates with metadata (issuer, expiry, etc.)
    • Certificate Deletion: Remove certificates from truststore
    • Self-Signed Certificate Support: Register certificates for systems using self-signed TLS

    7. JVM Preferences and Tuning¶

    Previous Versions¶

    • CLI: Manual JVM arguments in startup scripts
    • Thymeleaf: No JVM configuration UI

    What's New¶

    • JVM Settings Panel: Configure JVM parameters through the UI:
    • Initial heap size (Xms)
    • Maximum heap size (Xmx)
    • Garbage collection settings
    • Additional JVM arguments
    • Persistent Settings: JVM settings stored in web.properties
    • Reset to Defaults: One-click restore of default settings
    • Restart Guidance: Clear indication when restart is required

    8. AI Agent Integration (Claude)¶

    Previous Versions¶

    • CLI: No AI/LLM features
    • Thymeleaf: No AI/LLM features

    What's New (AI-Powered Features)¶

    • Claude Agent Integration: AI analysis powered by Claude Code CLI with MCP tool access to HMS-Mirror's REST API
    • One-Click AI Analysis: Available across all major pages:
    • Configuration analysis and recommendations
    • Job configuration review
    • Migration report insights
    • Connection troubleshooting and diagnostics
    • Interactive Assistant Chat: Multi-tab chat interface with conversation persistence across page navigation
    • Chat Export: Download conversations as Markdown files
    • Session Management: Automatic session expiry with timeout banners and idle cleanup
    • Analysis Persistence: Analysis results survive page navigation via the analysis tracker service
    • Model Selection: Configure the Claude model from Preferences > AI Agent
    • Feature Flag: Single hms-mirror.features.llm.enabled flag controls all AI features
    • Docker Ready: Claude CLI pre-installed in the Docker image with ANTHROPIC_API_KEY passthrough

    9. Cloudera Platform Framework (CPF) Plugin¶

    Previous Versions¶

    • CLI: Standalone only
    • Thymeleaf: Standalone only

    What's New¶

    • CMA Embedded Mode: HMS-Mirror runs as a PF4J plugin inside Cloudera Manager Application (CMA)
    • Shared Authentication: Plugin inherits CMA's authentication and user context
    • DispatcherServlet Router: Lightweight Spring MVC router dispatches REST calls within the plugin classloader
    • MCP Tool Provider: Plugin exposes HMS-Mirror tools through CPF's native MCP integration for AI assistants
    • CM Proxy Client: Cloudera Manager API calls route through CPF's built-in proxy
    • Adaptive UI: Frontend adapts layout (removes header, adjusts sidebar) when running inside CMA
    • Independent Build: Plugin POM decoupled from reactor parent to avoid dependency conflicts

    10. Security Hardening¶

    Previous Versions¶

    • CLI: Relies on OS-level security
    • Thymeleaf: Basic authentication

    What's New¶

    • MCP Authentication: Per-user JWT tokens for MCP tool calls with 8-hour TTL
    • WebSocket Security: JWT-based handshake authentication for Socket.IO connections
    • API Key Management: Self-service API key creation from the Profile page
    • Account Lockout Protection: Rate-limiting login filter with configurable thresholds and auto-unlock cooldown
    • Admin Account Unlock: UI for viewing locked accounts and unlocking them
    • Swagger UI Gating: Disabled by default; requires ADMIN role when enabled
    • Configurable CORS: HMS_MIRROR_CORS_ALLOWED_ORIGINS for production deployments behind reverse proxies
    • JWT Secret Management: Admin UI for configuring persistent JWT secrets with environment variable support
    • Registration Control: Toggle open self-registration on or off

    11. CMA Visual Alignment¶

    Previous Versions¶

    • CLI: N/A
    • Thymeleaf: Custom styling

    What's New¶

    • CMA Design Tokens: Colors, typography, spacing, and border-radius aligned with Cloudera Manager Application
    • Roboto Font: Typography updated to match CMA's font family
    • CUIX Icons: Replaced Heroicons with Cloudera CUIX icon set throughout all components
    • CMA Layout Components: PageHeader, PageSubtitleBar, Breadcrumb, DashboardPanel, and FormField components
    • Dark Sidebar: Permanently dark navigation sidebar matching CMA style
    • Flattened Cards: Cards use CMA-style flat design without shadows or hover animations

    12. RocksDB Persistence¶

    Previous Versions¶

    • CLI: File-based configuration, no persistence between runs
    • Thymeleaf: Session-based storage, lost on restart

    What's New¶

    • Persistent Storage: All data stored in RocksDB embedded database
    • Column Families: Organized storage for different entity types:
    • Configurations
    • Connections
    • Datasets
    • Jobs
    • Conversion Results
    • Run Status
    • Preferences
    • Sessions
    • RocksDB Browser: Explore stored data with:
    • Column family navigation
    • Key-value pair viewing
    • Regex search support
    • Raw value display
    • Maintenance Operations:
    • Health monitoring
    • Statistics viewing
    • Data compaction
    • Backup creation and restoration
    • Data Integrity: Transaction-safe operations

    • CLI: File-based reports in output directory

    What's New¶

    • Reports Browser: Navigate report directory hierarchy

    • Detailed migration results

    • LLM Report Analysis: AI-powered insights on migration results

    • YAML Configuration Viewer: View configuration files inline

    11. Driver Management¶

    • HDP 2.x
    • CDP 7.1.9, 7.3.1
    • Metastore Drivers: Separate management for metastore database drivers (MySQL, PostgreSQL, Oracle, MariaDB)
    • Driver Verification: Check driver availability per connection

    Migration Path¶

    From CLI¶

    1. Start the HMS-Mirror web server
    2. Use Utilities > Import CLI Config to import existing configurations
    3. Review and adjust imported connections and settings
    4. Create jobs using imported configurations

    From Thymeleaf Version¶

    1. Install the new v4.0 web application
    2. Configurations will need to be recreated (RocksDB is a new storage backend)
    3. Use the wizards to recreate connections and jobs
    4. Take advantage of new features like persistent jobs and Kerberos management

    System Requirements¶

    • Java: 17 or higher
    • Memory: 4GB minimum, 8GB+ recommended
    • Disk: 10GB+ for RocksDB storage and reports
    • Browser: Modern browser (Chrome, Firefox, Safari, Edge)
    • Optional: Ollama for LLM features

    Getting Started¶

    1. Download and extract the HMS-Mirror distribution
    2. Run bin/hmsm-server to start the web server
    3. Access the UI at http://localhost:8090/hms-mirror
    4. Use the Configuration Wizard to create your first migration configuration
    5. Download required drivers via Utilities > Drivers
    6. Create connections using the Connection Wizard
    7. Define datasets and create jobs
    8. Execute and monitor migrations through the web interface

    For detailed documentation, see the HMS-Mirror Documentation.
