Running LLAP on Compute Only Nodes

Oct 15, 2019

To run LLAP and/or Hive jobs on Compute Only Nodes you need to override a Hive Configuration default setting. Otherwise, queries will fail a majority of the time.

The hive configuration value hive.merge.nway.joins is true by default. But in Ambari 2.7.3+ (HDP 3+), it overrides the default setting and configures this to false. When this is left at the default true, intermittent failures of queries will happen on compute only nodes.

In source @ HiveConf.java the default is true, so it needs to be set and overridden.

NWAYJOINREORDER("hive.reorder.nway.joins", true,
  "Runs reordering of tables within single n-way join (i.e.: picks streamtable)"),

So, for platforms with compute only nodes, set in hive config:

hive.merge.nway.join=false

#hive #compute only node# #llap #tez #config #nway #reorder #query failure#
